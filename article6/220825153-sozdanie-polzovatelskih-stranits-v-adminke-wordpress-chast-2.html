Создание страницы администрирования

Поскольку мы продолжаем с плагином в этой статье, вот то, что мы будем делать:

  1. Создадим разметку страницы администрирования, в естественном стиле WordPress.
  2. Введем параметр, с которым пользователь может взаимодействовать.
  3. Обработаем параметр перед его сохранением.
  4. Извлечём значение параметра и зададим ему значение, равное значению на странице.

В этом уроке мы сосредоточимся на следующих двух моментах. В следующей статье мы сосредоточимся на моментах 3 и 4.

Для тех, кто знаком с API Настроек, это кажется излишним. Но, как вы увидите, по оставшейся части учебника из этой серии, есть причина, почему мы собираемся разбить его на более мелкие части, как этот.

План действий мы изложили, давайте начнем.

Разметка страницы

Когда мы в прошлый раз ушли от этой функции, код выглядел следующим образом:

<?phpclass Submenu_Page {    /**     * This function renders the contents of the page associated with the Submenu     * that invokes the render method. In the context of this plugin, this is the     * Submenu class.     */    public function render() {        echo "This is the basic submenu page.";    }}

Теперь мы действительно готовы к созданию разметки страницы. Всякий раз, когда я работаю над плагинами для клиентов, я называю это «представления» ("views").

Только не нужно путать с парадигмой модель-представление-контроллер (MVC). И я не называю их шаблонами потому, что в WordPress шаблоны используются для отображения содержимого страницы.

Итак, это будут "представления".

Первое, что мы хотим сделать это создать папку представлений внутри папки admin, нашего плагина.

Как только это будет сделано, мы можем назвать это как-нибудь просто, например settings.php или как-нибудь более образно. Это решать вам, учитывая сложность того, что вы хотите сделать. Так как это простой плагин, я оставлю простое название.

Далее, давайте начнем с создания основной разметки, которая предусматривает стандартный элемент обертки WordPress, наряду с заголовком страницы:

<div class="wrap">    <h1><?php echo esc_html( get_admin_page_title() ); ?></h1>    <form method="post" action="<?php echo esc_html( admin_url( "admin-post.php" ) ); ?>">    </form></div><!-- .wrap -->

Здесь следует отметить, что заголовок предоставляется от функции, которая использует значения, передаваемые в функцию add_options_page, с которой мы работали еще на начальном этапе нашей работы над плагином. Дальше, мы используем функцию admin_url для обозначения, где мы будем размещать параметры (о которых мы поговорим чуть позже).

И в обоих случаях, мы используем функцию очистки esc_html чтобы быть уверенными, что вредоносные строки не смогут отображаться на странице.

Добавим параметр

На данном этапе, мы уже готовы добавить параметр. По назначению данного поста, мы разрешим пользователю ввести что-то в поле ввода текста. Это позволит нам взглянуть на то, как обезопасить информацию и в итоге показать её на странице.

Для этого нам потребуется ключевой фрагмент информации. А именно, мы должны знать название атрибута, которым мы назовем элемент input. Это для того, чтобы можно было сохранить его в базе данных должным образом.

Таким образом, для целей этого примера, пусть это сообщение будет отображаться в верхней части каждой записи. Мы не дойдем до сути в этой статье, но мы пересмотрим это в следующей статье.

В вашем settings.php добавьте следующий код.

<div class="wrap">    <h1><?php echo esc_html( get_admin_page_title() ); ?></h1>  <form method="post" action="<?php echo esc_html( admin_url( "admin-post.php" ) ); ?>"><div id="universal-message-container"><h2>Universal Message</h2><div class="options"><p><label>What message would you like to display above each post?</label><br /><input type="text" name="acme-message" value="" /></p></div><!-- #universal-message-container --></form></div><!-- .wrap -->

Если вы посмотрите страницу на данной стадии, вы увидите именно то, что ожидается: ярлык и поле ввода.

Обезопасим наш Параметр

Обычно, я бы показал здесь скриншот, но мы еще не закончили. Нам нужно ввести еще две вещи.

  1. WordPress nonce
  2. Кнопку отправки

Давайте сделаем это сейчас. Если вы не знакомы со значением nonce, я рекомендую прочитать эту статью. Коротко говоря:

Nonce — это «однократно используемое число» для защиты URL и форм от определенного вида неправильного или вредоносного использования и тому подобное.

Добавим следующий блок кода в наш файл settings.php:

<?php    wp_nonce_field( "acme-settings-save", "acme-custom-message" );    submit_button();?>

Окончательный вариант кода должен выглядеть следующим образом:

<div class="wrap">    <h1><?php echo esc_html( get_admin_page_title() ); ?></h1><form method="post" action="<?php echo esc_html( admin_url( "admin-post.php" ) ); ?>"><div id="universal-message-container"><h2>Universal Message</h2><div class="options"><p><label>What message would you like to display above each post?</label><br /><input type="text" name="acme-message" value="" /></p></div><!-- #universal-message-container --><?phpwp_nonce_field( "acme-settings-save", "acme-custom-message" );submit_button();?></form></div><!-- .wrap -->

Теперь мы готовы сохранить параметры. И чтобы сделать это, нам нужно проверить пару вещей:

  1. Есть ли у пользовтаеля разрешение на сохранение параметров?
  2. Действительный ли nonce передается из формы?

Кроме того нам необходимо:

  1. Обезопасить информацию, чтобы убедиться, что вредоносные данные не попадут в базу данных.
  2. Мы должны суметь вытащить эту информацию из базы данных и заполнить её в форму.

И наконец, нам нужно отобразить это на нашем веб-сайте.

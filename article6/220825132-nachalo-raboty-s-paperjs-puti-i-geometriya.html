Работа с путями

Пути в Paper.js представлены последовательностью сегментов, которые связаны кривыми. Сегмент в основном представляет собой point и две ее ручки, которые определяют местоположение и направление кривых. Если ручки не определены, то это приведет к прямым линиям вместо кривых.

Когда вы определяете новый путь с помощью конструктора new Path(), вы можете добавлять к нему сегменты с помощью функции path.add(segment). Поскольку эта функция поддерживает несколько аргументов, вы также можете добавить сразу несколько сегментов. Предположим, вы хотите вставить новый сегмент по определенному индексу внутри существующего пути. Вы можете сделать это, используя функцию path.insert(index, segment). Аналогично, чтобы удалить сегмент с определенным индексом, вы можете использовать функцию path.removeSegment(index). Обе эти функции используют индексирование с нулевым индексом. Это означает, что использование path.removeSegment(3) удалит четвертый сегмент. Вы можете закрыть все пути, которые вы рисуете, с помощью свойства path.closed. Он соединит первый и последний сегменты пути вместе.

До сих пор все наши пути имели прямые линии. Чтобы создать кривые пути без указания дескрипторов для каждого сегмента, вы можете использовать функцию path.smooth(). Эта функция вычисляет оптимальные значения для ручек всех сегментов в пути таким образом, что проходящая через них кривая является гладкой. Сегменты сами по себе не меняют свое местоположение, и если вы указали значения дескриптора для любого из сегментов, эти значения будут проигнорированы. В приведенном ниже коде используются все функции и свойства, которые мы обсуждали, для создания четырех путей, два из которых являются изогнутыми.

var aPath = new Path();aPath.add(new Point(30, 60));aPath.add(new Point(100, 200));aPath.add(new Point(300, 280), new Point(280, 40));aPath.insert(3, new Point(180, 110));aPath.fullySelected = "true";aPath.closed = true;  var bPath = aPath.clone();bPath.smooth();bPath.position.x += 400;  var cPath = aPath.clone();cPath.position.y += 350;cPath.removeSegment(3);  var dPath = bPath.clone();dPath.strokeColor = "green";dPath.position.y += 350;dPath.removeSegment(3);

Сначала мы создаем новый путь, а затем добавляем к нему сегменты. Использование path.insert(3, new Point (180, 110)) вставляет новый сегмент вместо четвертого и перемещает четвертый сегмент в пятую позицию. Я установил fullSelected в true, чтобы показать вам все точки и ручки для каждой кривой. Для второго пути я использовал функцию path.smooth(), чтобы сделать кривую гладкой. Удаление четвертого сегмента с помощью cPath.removeSegment(3) дает нам нашу оригинальную форму без каких-либо индексированных вставок. Вы можете проверить это, комментируя aPath.insert(3, new Point(180, 110)); в этой демо-версии CodePen. Это конечный результат всех наших манипуляций до этого момента:

Предопределенные фигуры

Paper.js поддерживает некоторые основные формы из коробки. Например, чтобы создать круг, вы можете просто использовать новый конструктор Path.Circle(center, radius). Аналогичным образом вы можете использовать конструктор new Path.Rectangle(rect) для создания прямоугольника. Вы можете указать верхний левый и нижний правый углы или указать верхний левый угол и размер прямоугольника. Чтобы нарисовать прямоугольник с закругленными углами, вы можете использовать конструктор new Path.RoundedRectangle(rect, size), где параметр размера определяет размер закругленных углов.

Если вы хотите создать n-сторонний регулярный многоугольник, вы можете сделать это, используя конструктор new Path.RegularPolygon(center, numSides, radius). Параметр center определяет центр нашего многоугольника, а радиус определяет радиус нашего многоугольника.

В приведенном ниже коде будут генерироваться все формы, которые мы только что обсуждали.

var aCircle = new Path.Circle(new Point(75, 75), 60);aCircle.strokeColor = "black";  var aRectangle = new Path.Rectangle(new Point(200, 15), new Point(400, 135));aRectangle.strokeColor = "orange";  var bRectangle = new Path.Rectangle(new Point(80, 215), new Size(400, 135));bRectangle.strokeColor = "blue";  var myRectangle = new Rectangle(new Point(450, 30), new Point(720, 170));var cornerSize = new Size(10, 60);var cRectangle = new Path.RoundRectangle(myRectangle, cornerSize);cRectangle.fillColor = "lightgreen";  var aTriangle = new Path.RegularPolygon(new Point(120, 500), 3, 110);aTriangle.fillColor = "#FFDDBB";aTriangle.selected = true;var aDodecagon = new Path.RegularPolygon(new Point(460, 490), 12, 100);aDodecagon.fillColor = "#CCAAFC";aDodecagon.selected = true;

Первый прямоугольник, который мы создаем, основан на координатных точках. Следующий использует первую точку для определения верхнего левого угла нашего прямоугольника, а затем использует значение размера для построения остальных точек. В третьем прямоугольнике мы дополнительно указали радиус для нашего прямоугольника. Первый параметр радиуса решает горизонтальную кривизну, а второй параметр определяет вертикальную кривизну.

Последние две формы просто используют конструктор RegularPolygon для создания треугольника и додекагона. Введенная демонстрация ниже показывает результат нашего кода.

Упрощение и выравнивание путей

Существует два способа создания круга. Первый заключается в создании множества сегментов без каких-либо ручек и их непосредственной близости. Таким образом, хотя они будут соединены прямой линией, общая форма все равно будет ближе к кругу. Второй способ - использовать только четыре сегмента с соответствующими значениями для их ручек. Это может сэкономить массу памяти и дать нам желаемые результаты.

Большую часть времени мы можем удалить довольно много сегментов из пути без существенных изменений в его форме. Библиотека обеспечивает простую функцию path.simplify([tolerance]) для достижения этого результата. Параметр допуска не является обязательным. Он используется для указания максимального расстояния, на которое алгоритм упрощения пути может отклоняться от исходного пути. По умолчанию установлено значение 2.5. Если вы установите параметр на более высокое значение, конечная кривая будет немного более гладкой, с меньшим количеством сегментов, но отклонение может быть значительным. Аналогичным образом, меньшее значение приведет к очень малым отклонениям, но включает в себя еще много сегментов.

Вы также можете преобразовать кривые в путь к прямым линиям, используя функцию path.flatten(maxDistance). При выравнивании пути библиотека пытается максимально увеличить расстояние между сегментами как можно ближе к maxDistance.

var aPolygon = new Path.RegularPolygon(new Point(140, 140), 800, 120);aPolygon.fillColor = "#CCAAFC";aPolygon.selected = true;  var bPolygon = aPolygon.clone();bPolygon.fillColor = "#CCFCAA";bPolygon.simplify();  var cPolygon = aPolygon.clone();cPolygon.fillColor = "#FCAACC";cPolygon.simplify(4);  var dPolygon = bPolygon.clone();dPolygon.fillColor = "#FCCCAA";dPolygon.flatten(80);

В приведенном выше коде я сначала создал многоугольник, используя описанную выше функцию RegularPolygon. Я намеренно установил для свойства selected значение true, чтобы все сегменты с этих путей были видны. Затем я клонировал второй многоугольник из первого и использовал на нем функцию simplify. Это уменьшило количество сегментов до пяти.

В третьем многоугольнике я установил параметр допуска к более высокому значению. Это дополнительно уменьшает количество сегментов. Вы можете видеть, что все пути все еще имеют одинаковую базовую форму. В конечном пути я использовал функцию flatten(maxDistance), чтобы сгладить нашу кривую. Алгоритм пытается сохранить форму как можно ближе к исходной, по-прежнему соблюдая ограничение maxDistance. Вот окончательный результат:

Геометрия и математика

Paper.js имеет некоторые базовые типы данных, такие как Point, Size и Rectangle для описания геометрических атрибутов графических элементов. Они представляют собой абстрактные представления геометрических значений, таких как местоположение или измерение. Точка просто описывает двумерное местоположение, а размер описывает абстрактные измерения в двумерном пространстве. Прямоугольник здесь обозначает область, заключенную в верхнюю левую точку, ее ширину и высоту. Он отличается от прямоугольного пути, который мы обсуждали ранее. В отличие от пути, это не элемент. Вы можете прочитать о них все в этом учебнике Paper.js.

Вы можете выполнять основные математические операции - сложение, вычитание, умножение и деление на точки и размеры. Все действия, приведенные ниже, действительны:

var pointA = new Point(20, 10);var pointB = pointA * 3; // { x: 60, y: 30 }var pointC = pointB - pointA; // { x: 40, y: 20 }var pointD = pointC + 30; // { x: 70, y: 50 }var pointE = pointD / 5;  // { x: 14, y: 10 }var pointF = pointE * new Point(3, 2); // { x: 42, y: 20 }// You can check the output in console for verificationconsole.log(pointF);

Помимо этих основных операций вы также можете выполнять некоторые операции округления или генерировать случайные значения для точек и размеров. Рассмотрим следующие примеры:

var point = new Point(3.2, 4.7);var rounded = point.round(); // { x: 3, y: 5 }var ceiled  = point.ceil();  // { x: 4, y: 5 }var floored = point.floor(); // { x: 3, y: 4 }// Generate a random point with x between 0 and 50// and y between 0 and 40var pointR = new Point(50, 40) * Point.random();// Generate a random size with width between 0 and 50// and height between 0 and 40var sizeR = new Size(50, 40) * Size.random();

Функция random() генерирует случайные значения от 0 до 1. Вы можете умножить их на объект Point или Size соответствующего значения, чтобы получить желаемые результаты.

Это завершает основную математику, которую вам нужно знать, чтобы создать что-то полезное с Paper.js.

Начало работы с Paper.js: Проекты и элементы...

Интро

Возможно, вы слышали о Paper.js. Итак, давайте начнем с вопроса: что такое Paper.js? Это библиотека, которая позволяет создавать и работать с векторной графикой. Официальный сайт описывает это как швейцарский армейский нож для создания векторной графики. ...

Возможно, вы слышали о Paper.js. Итак, давайте начнем с вопроса: что такое Paper.js? Это библиотека, которая позволяет создавать и работать с векторной графикой. Официальный сайт описывает это как швейцарский армейский нож для создания векторной графики.

Несмотря на то, что библиотека может многое предложить, ее легко узнать, даже если вы никогда не слышали о ней раньше. В этом уроке я начну с самых основ библиотеки, а затем перейду к сложным темам позже.

Использование PaperScript

Существует два способа использования библиотеки. Вы можете использовать PaperScript, который является расширением JavaScript, или просто использовать простой старый JavaScript.

PaperScript - это тот же старый JavaScript, который вы всегда использовали. Однако он добавляет поддержку математических операторов для объектов point и size. Это также упрощает установку обработчиков событий для объектов Project, View и Tool.

При загрузке PaperScript вы должны использовать обычный тег скрипта с типом, установленным на «text/paperscript». Если вы загружаете код извне, вам также нужно добавить тег <script> с соответствующим URL-адресом для загрузки кода. Одним из последних атрибутов, которые необходимо указать, является data-paper-canvas = "canvasId", или сокращенная версия canvas="canvasId", которая сообщает библиотеке о холсте, над которым она должна работать. Следующий код создает четырехугольник в PaperScript.

<script type="text/paperscript" canvas="quad">  var path = new Path();  path.strokeColor = "black";  var pointOne     = new Point(100, 20);  var pointTwo     = new Point(-100, 100);  var pointThree   = new Point(300, 30);  path.moveTo(pointOne);  path.lineTo(pointOne + pointTwo);  path.lineTo(pointTwo + pointThree);  path.lineTo(pointOne + pointThree);  path.closed = true;</script>

Использование JavaScript

Если вам не нравится PaperScript, вы также можете использовать JavaScript в своих проектах. Вам придется добавить еще несколько строк кода, если вы решите сделать это таким образом. Первое, что вам нужно сделать, это проверить, готов ли DOM, потому что до этого вы не сможете работать с холстом. После этого вы можете настроить проект и представление с использованием объекта paper. Все классы и объекты Paper.js теперь будут доступны только через объект paper.

Как я уже указывал ранее, при работе с точками и размерами вам придется использовать математические функции вместо операторов. Приведенный ниже код иллюстрирует все эти различия:

window.onload = function() {  var canvas = document.getElementById("quad");  paper.setup(canvas);  var path = new paper.Path();  path.strokeColor = "black";  var pointOne     = new paper.Point(100, 20);  var pointTwo     = new paper.Point(-100, 100);  var pointThree   = new paper.Point(300, 30);  path.moveTo(pointOne);  path.lineTo(pointOne.add(pointTwo));  path.lineTo(pointTwo.add(pointThree));  path.lineTo(pointOne.add(pointThree));  path.closed = true;  paper.view.draw();}

Как видно из приведенных выше фрагментов кода, при работе с Paper.js относительно легко использовать PaperScript. Поэтому все примеры с этого момента будут основаны на PaperScript.

Иерархия проектов

Если вы когда-либо использовали графическое приложение, такое как Adobe Photoshop или Illustrator, вы должны быть знакомы с концепцией слоев. Каждый слой в этих программах имеет свой собственный контент, который в сочетании с другими слоями создает конечный результат. Аналогичные слои также существуют в Paper.js и могут быть доступны с помощью project.layers.

Первоначально каждый проект имеет один слой, доступный через project.activeLayer. Любые новые элементы, которые вы создаете, добавляются к текущему активному слою в качестве его дочернего элемента. Доступ ко всем дочерним элементам в определенном слое можно с помощью свойства layer.children активного слоя.

Существует несколько способов доступа ко всем этим детям. Если вам нужен только доступ к первому и последнему ребенку любого элемента, вы можете использовать item.firstChild и item.lastChild соответственно. Вы также можете назначить определенное имя любому ребенку, а затем использовать это имя для доступа к нему позже. Скажем, слой, на котором вы работаете, составляет около 30 детей. Нецелесообразно перебирать их по одному. По этой причине в библиотеке есть свойство layer.children.length, которое вы можете использовать для получения общего количества дочерних элементов, а затем итерации по списку с использованием цикла for.

Этот фрагмент кода обращается к различным дочерним элементам, используя все свойства, которые мы только что обсуждали:

var circleA = new Path.Circle(new Point(45, 150), 45);var circleB = new Path.Circle(new Point(110, 150), 20);var circleC = new Path.Circle(new Point(165, 150), 35);var circleD = new Path.Circle(new Point(255, 150), 55);var circleE = new Path.Circle(new Point(375, 150), 65);var circleF = new Path.Circle(new Point(475, 150), 35);circleC.name = "GreenCircle";project.activeLayer.firstChild.fillColor = "orange";project.activeLayer.lastChild.fillColor = "pink";project.activeLayer.children[1].fillColor = "purple";project.activeLayer.children["GreenCircle"].fillColor = "lightgreen";for (var i = 3; i < 5; i++) {  project.activeLayer.children[i].fillColor = "tomato";}

Введенная демонстрация ниже показывает сценарий в действии. Вы можете убедиться, что цвет всех кругов соответствует цвету, который мы назначили им в приведенном выше коде.

Вы также можете использовать метод item.parent для доступа к родительскому элементу элемента, например метода item.children, к которому вы использовали доступ ко всем своим дочерним элементам. Всякий раз, когда вы создаете новый элемент, его родительский элемент всегда будет активным в настоящий момент уровнем проекта. Однако его можно изменить, добавив элемент как дочерний элемент другого layer или group.

Прежде чем идти дальше, позвольте мне объяснить, что такое group на самом деле. Честно говоря, и layer, и group очень похожи. Одно из основных различий между этими двумя заключается в том, что любые новые элементы, которые вы создаете, автоматически добавляются в активный слой, но в случае группы вам придется добавлять элементы самостоятельно.

Существует несколько способов добавления элементов в группу. Вы можете передать элемент массивов в конструктор группы, и все они будут добавлены в массив item.children группы. Чтобы добавить элементы в группу после ее создания, вы можете использовать функцию item.addChild(item). Вы также можете вставить ребенка по определенному индексу, используя функцию item.insertChild(index, item).

Удаление элементов также так же просто, как и их добавление. Чтобы удалить любой объект из проекта, вы можете использовать функцию item.remove(). Имейте в виду, что это не уничтожит элемент, и его можно будет добавить обратно в проект в любое время. Если у предмета, который вы удалили, были дети, все дети также будут удалены. Что делать, если вы хотите удалить всех детей, но сохраните неповрежденный предмет? Этого можно добиться с помощью функции item.removeChildren().

Понимание элементов

Элемент iterm появился более чем несколько раз в учебнике. Итак, что это? Все, что появляется в проекте Paper.js, является item. Сюда входят layers, paths, groups и т.д. В то время как разные элементы имеют свойства, характерные для них, другие свойства применимы ко всем из них.

Если вы намерены скрыть элемент от пользователя, вы можете сделать это, установив значение item.visible в false. Вы также можете клонировать любой элемент, используя функцию item.clone(). Эта функция возвращает клонированный элемент, который вы можете сохранить в переменной и манипулировать позже. Вы также можете изменить прозрачность любого элемента, используя свойство item.opacity. Любое значение между 0 и 1 сделает элемент полупрозрачным.

Вы также можете установить режим смешивания для любого элемента, используя свойство item.blendMode. Режим смешивания должен быть передан как string. Библиотека также предоставляет свойство item.selected, которое, если установлено значение true, создает визуальный контур поверх этого элемента. Это может быть очень полезно при отладке, так как позволяет просматривать контуры, отдельные точки сегмента и ограничивающие прямоугольники элементов.

Преобразование предметов

Элементы можно легко масштабировать, поворачивать или перемещать в проекте Paper.js. В этом разделе я кратко расскажу обо всех этих преобразованиях.

Чтобы изменить положение item, вы можете использовать его свойство item.position и установить позицию в новую точку. Если вы хотите перемещать элемент вокруг, вы можете сделать это с помощью оператора +=.

Вы также можете масштабировать любой элемент, используя функцию item.scale(scale). Это будет масштабировать элемент вокруг его центральной точки. Вы можете масштабировать элемент вокруг какой-либо другой точки, указав ее как второй параметр, например item.scale(scale, point). Кроме того, библиотека также позволяет масштабировать объекты по-разному в вертикальном и горизонтальном направлениях, передавая два числа в качестве параметров, например item.scale(scaleX, scaleY).

Вращающиеся элементы похожи на их масштабирование. Вы можете использовать функцию item.rotate(angle) для поворота элементов вокруг их центра. Угол задается в градусах, а вращение происходит по часовой стрелке. Чтобы повернуть элемент вокруг определенной точки, вы также можете передать точку в качестве второго параметра, например item.rotate(angle, point).

Следующий фрагмент кода применяет все преобразования и манипуляции, которые мы обсуждали только в трех разных прямоугольниках.

var rectA = new Path.Rectangle(new Point(250, 70), new Size(120, 120));rectA.fillColor = "pink";var rectB = rectA.clone();rectB.fillColor = "purple";rectB.position += new Point(80, 80);rectB.opacity = 0.6;rectB.blendMode = "color-burn";rectB.scale(1.5);rectB.rotate(45);var rectC = rectB.clone();rectC.fillColor = "lightgreen";rectC.position += new Point(-180, 0);rectC.blendMode = "color-dodge";rectC.scale(1.5);

Код в значительной степени не требует пояснений. Я клонировал прямоугольник B из прямоугольника A, а прямоугольник B приобретает все свойства прямоугольника A. То же самое касается прямоугольника B и C.

Обратите внимание, что я использовал оператор +=, о котором я говорил выше, чтобы перемещать элементы вокруг. Этот оператор перемещает элементы относительно своих старых позиций вместо использования абсолютных значений.

Определение значений свойств

Anime.js позволяет указать конечные значения для анимационных свойств целевых элементов. Начальное значение для анимации является значением по умолчанию этого свойства. Любое значение, указанное в CSS, также может служить стартовым значением. Существует несколько способов указания конечного значения.

Оно также может быть безразмерным числом. В этом случае исходные или стандартные единицы свойства используются при вычислении любых значений свойств. Вы также можете указать значение как строку, но строка должна содержать хотя бы одно числовое значение. Примерами строковых значений будут 10vh, 80% и 9.125turn.

Вместо указания абсолютного значения вы также можете указать значения свойств относительно их текущего значения. Например, вы можете установить конечное значение translateY на 150px больше, чем текущее значение, используя значение +=150px в качестве значения. Имейте в виду, что вы можете использовать только добавление, умножение и вычитание при определении относительных значений.

При анимации цветов вы не можете использовать названия цветов, такие как красный, черный и синий, чтобы установить конечное значение цвета для анимации. В таких случаях цветовой анимации не будет вообще, и изменение будет мгновенным. Единственный способ анимации цветов - указать значения как шестнадцатеричные цифры, или значения RGB и HSL.

Как вы могли заметить, мы не указали начальное значение для наших целевых элементов для их анимирования. Anime.js автоматически определяет начальное значение на основе нашего CSS и значений по умолчанию для этих свойств. Однако вы можете указать начальное значение, отличное от значения по умолчанию для свойства с использованием массивов. Первый элемент в массиве означает начальное значение, а второй элемент означает конечное значение.

Вместо использования того же конечного значения для всех ваших целевых элементов вы можете использовать функции для установки разных значений для разных параметров. Этот процесс аналогичен заданию параметров свойств на основе функций.

var uniqueTranslation = anime({  targets: ".square",  translateY: function(el, i) {    return 50 * (i + 1);  },  autoplay: false});var randomScaling = anime({  targets: ".square",  scale: function(el, i) {    return Math.random()*1.5 + i/10;  },  autoplay: false});var randomRotation = anime({  targets: ".square",  rotate: function() {    return anime.random(-180, 180);  },  autoplay: false});var randomRadius = anime({  targets: ".square",  borderRadius: function(el) {    return 20 + Math.random()*el.offsetWidth/4;  },  autoplay: false});var randomAll = anime({  targets: ".square",  translateY: function(el, i) {    return 50 + 50 * i;  },  scale: function(el, i) {    return Math.random()*1.5 + i/10;  },  rotate: function() {    return anime.random(-180, 180);  },  borderRadius: function(el) {    return Math.random()*el.offsetWidth/2;  },  duration: function() { return anime.random(1500, 2400); },  delay: function() { return anime.random(0, 1000); },  autoplay: false});

Для свойства translateY мы используем индекс элемента для установки значения перевода. Использование 50 * (i + 1) увеличивает значение translateY для каждого элемента на 50 пикселей.

Масштабирующая анимация также использует индекс элемента вместе со встроенной функцией Math.random(), чтобы возвращать псевдослучайное число с плавающей запятой меньше 1. Таким образом, элементы масштабируются случайным образом, но часть i/10 свойства немного увеличивает вероятность элементов, которые в конечном итоге имеют больший размер.

Внутри кода для анимации вращения мы используем вспомогательную функцию anime.random(a, b) для получения случайных целых чисел от -180 до 180. Эта функция полезна для назначения случайных интегральных значений таким свойствам, как translateY и rotate. Использование этой функции для назначения случайных значений шкалы даст экстремальные результаты.

Значение радиуса границы для разных элементов определяется путем вычисления ширины целевых элементов с использованием параметра функции el. Наконец, последняя часть кода также назначает случайные значения параметрам duration и delay.

Вы можете видеть, что анимация, достигнутая последней частью, очень случайна. Нет никакой связи между значениями различных свойств элементов или их значениями задержки и продолжительности. В реальной жизни более разумно использовать значения, которые могут добавить некоторое чувство направления в анимацию.

Также возможно анимировать различные свойства ваших целевых элементов, используя ключевые кадры. Каждый ключевой кадр состоит из массива c объектом свойств. Объект можно использовать для указания значения свойства, duration, delay и easing для этой части анимации. Следующий код создает анимацию трансляции на основе ключевых кадров.

var keyframeTranslation = anime({  targets: ".square",  translateY: [    { value: 100, duration: 500},    { value: 300, duration: 1000, delay: 1000},    { value: 40, duration: 500, delay: 1000}  ],  autoplay: false});var keyframeAll = anime({  targets: ".square",  translateY: [    { value: 100, duration: 500},    { value: 300, duration: 1000, delay: 1000},    { value: 40, duration: 500, delay: 1000}  ],  scale: [    { value: 1.1, duration: 500},    { value: 0.5, duration: 1000, delay: 1000},    { value: 1, duration: 500, delay: 1000}  ],  rotate: [    { value: 60, duration: 500},    { value: -60, duration: 1000, delay: 1000},    { value: 75, duration: 500, delay: 1000}  ],  borderRadius: [    { value: 10, duration: 500},    { value: 50, duration: 1000, delay: 1000},    { value: 25, duration: 500, delay: 1000}  ],  delay: function(el, i) { return 100*(i+1) },  autoplay: false});

Вы также можете анимировать несколько свойств сразу, указав разные или одинаковые значения для всех параметров. Во втором случае глобальный параметр delay применяет начальную задержку ко всем элементам на основе их индекса. Эта задержка не зависит от задержки, применяемой к каждому свойству внутри ключевых кадров.

Создание и управление временными шкалами

До сих пор в серии мы использовали параметр delay для воспроизведения различных анимаций в определенной последовательности. Чтобы использовать задержку для этой цели, нам также нужно знать продолжительность предыдущей анимации.

С увеличением сложности последовательности анимации поддержание правильного значения задержки становится очень утомительным. Любое изменение длительности одной из анимаций заставит нас пересчитать все значения задержки, чтобы сохранить анимацию в исходной последовательности.

Лучшим решением этой проблемы является использование временных шкал для управления последовательностью анимации. Вы должны использовать anime.timeline() функцию для создания временной шкалы в Anime.js. Вы можете также передавать различные параметры этой функции в качестве объекта. Эти параметры могут указывать направление воспроизведения временной шкалы, число циклов и параметр autoplay , чтобы определить, должна ли анимация автоматически воспроизводиться. Все эти параметры подробно обсуждались в руководстве по параметрам этой серии.

Вы можете добавить разные анимации к временной шкале с помощью метода add(). Все анимации, добавленные к временной шкале, будут воспроизводиться в том порядке, в котором они были добавлены. Можно указать абсолютные или относительные значения смещения для управления порядком воспроизведения анимаций.

Когда используются относительные значения смещения, время начала текущей анимации определяется относительно времени предыдущей анимации. Относительные смещения могут быть трех типов:

  • +=offset: В этом случае текущая анимация начинает воспроизводиться после того, как offset количество миллисекунд прошло с конца предыдущей анимации.
  • -=offset: в этом случае текущая анимация начинает играть offset количество миллисекунд до конца предыдущей анимации.
  • *=offset: в этом случае текущая анимация начинает проигрываться через количество миллисекунд, равное offset времени, в течение которого прошла предыдущая анимация.

Следующий код показывает, как создать базовую шкалу времени и временную шкалу с относительными значениями смещения.

var basicTimeline = anime.timeline({  direction: "alternate",  loop: 2,  autoplay: false});basicTimeline.add({    targets: ".square",    translateY: 200 }).add({    targets: ".red",    translateY: 100 }).add({    targets: ".blue",    translateY: 0 });var offsetTimeline = anime.timeline({  direction: "alternate",  loop: 2,  autoplay: false});offsetTimeline.add({    targets: ".square",    translateY: 200 }).add({    targets: ".red",    offset: "+=1000",    translateY: 100 }).add({    targets: ".blue",    offset: "*=2",    translateY: 0 });

Попробуйте щелкнуть по кнопке Offset Timeline в приведенной выше демонстрации. Вы увидите, что между окончанием анимации красных квадратов и началом анимации синих квадратов существует задержка в 2 секунды.

Мы не указали duration анимации красного квадрата. Поэтому в качестве продолжительности используется значение по умолчанию 1000 мс или 1 с. Множественное смещение синей квадратной анимации удваивает это значение, и это приводит к задержке анимации в две секунды.

Когда используются абсолютные значения смещения, время начала временной шкалы используется в качестве контрольной точки. Можно изменить последовательность, в которой воспроизводятся анимации, используя большие значения смещения для анимаций, которые происходят в начале временной шкалы.

Параметры воспроизведения

Anime.js имеет множество возможностей для воспроизведения, паузы, перезапуска или поиска анимаций или временных шкал в любой момент.

Функция play() позволяет начать анимацию с ее текущего прогресса. Функция pause() заморозит анимацию в момент вызова функции. Функция restart() запускает анимацию с самого начала, независимо от ее текущего прогресса. Функция seek(value) может использоваться для продвижения анимации по значению value в миллисекундах.

Вы должны иметь в виду, что функция play() только возобновляет анимацию с момента ее приостановки. Если анимация уже достигла конца, вы не сможете воспроизвести анимацию с помощью play(). Чтобы воспроизвести анимацию, вам придется использовать функцию restart().

var slowAnimation = anime({  targets: ".square",  translateY: 250,  borderRadius: 50,  duration: 4000,  easing: "linear",  autoplay: false});document.querySelector(".play").onclick = slowAnimation.play;document.querySelector(".pause").onclick = slowAnimation.pause;document.querySelector(".restart").onclick = slowAnimation.restart;var seekInput = document.querySelector(".seek");seekInput.oninput = function() {  slowAnimation.seek(slowAnimation.duration * (seekInput.value / 100));};

Обратите внимание, что мы не используем seekInput.value для установки значения для функции поиска. Это связано с тем, что максимальное значение для ввода диапазона было установлено равным 100 в разметке. Непосредственно использование значения для входного диапазона позволит нам искать только до 100 мс. Умножая входное значение диапазона на продолжительность анимации, мы можем искать анимацию от начала до конца на нашем ползунке диапазона.

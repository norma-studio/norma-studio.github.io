Дизайн, использующий длинные тени, - это вариация плоского дизайна, к которому добавлены тени, что создает ощущение глубины и приводит к тому, что дизайн воспринимается как трехмерный. В этом уроке мы напишем плагин jQuery, который позволит нам легко преобразовывать плоский значок путем добавления к нему полностью настраиваемой длинной тени.

Если вас интересует полная версия jQuery плагина для добавления длинных теней к значкам и тексту, то посмотрите Long Shadow jQuery Plugin на CodeCanyon.

В этом уроке мы рассмотрим элементы дизайна с длинными тенями, а также мы напишем простой jQuery плагин, который позволит нам управлять этими элементами.

Давайте начнем!

Какие элементы используются в дизайне с длинными тенями?

Давайте определим, какие элементы используются для создания дизайна с длинными тенями. Мы можем выделить следующие:

  • Основной элемент, или элемент, который отбрасывает тень.
  • Длина тени, обычно очень длинная, из-за чего этот эффект и имеет такое название. Длина тени также создает ощущение того, что основной элемент имеет глубину.
  • Направление или угол тени. В реальном мире этот параметр определяется расположением источника света. Как правило, все тени, происходящие от одного источника света, имеют одинаковое направление.
  • Цвет и прозрачность тени. Цвет источника света влияет на цвет теней. Кроме того, чем интенсивнее источник света, тем темнее и четче тени.

Когда эти элементы используются вместе, они создают ощущение, что основной элемент не плоский, а трехмерный объект, который отбрасывает тень.

Итак, давайте теперь напишем jQuery плагин, который позволит нам контролировать вышеперечисленные элементы.

Пишем jQuery плагин для эффекта длинных теней

Написание jQuery плагина для длинных теней, мы начнем с создания базовой структуры проекта для плагина jQuery, вроде вот такой:

  • Создадим папку, в которой будут располагаться файлы нашего проекта. Назовем эту папку long-shadows-jquery-plugin.
  • Внутри папки нашего проекта создадим файл под названием index.html. В нем будет располагаться наш HTML код.
  • Создадим файл под названием jquery.longshadows.js , и поместим его в папку нашего проекта. В нем будет располагаться код JavaScript нашего jQuery плагина.
  • Для того, чтобы не смешивать разные части проекта, создадим еще один JavaScript файл в той же папке, назовем его script.js. Он нам нужен для того, чтобы использовать наш новый jQuery плагин в проекте.
  • Также поместим в папку проекта файл значка heart.png, который вы можете найти в прикрепленных к этому уроку файлах.

Наш файл index.html будет содержать базовую HTML разметку и включать jQuery и наши JavaScript файлы. Нам необходимо включить библиотеку jQuery, потому что мы будет использовать jQuery плагин. Содержимое файла index.html должно выглядеть вот так:

<html><head>    <script type="text/javascript" src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.2.1/jquery.min.js"></script>    <script type="text/javascript" src="jquery.longshadows.js"></script>    <script type="text/javascript" src="script.js"></script></head><body>    <img src="heart.png" id="logo"></body></html>

Файл jquery.longshadows.js будет содержать код jQuery плагина, который мы напишем следующим образом:

(function($) {    $.fn.longshadows = function(options) {        var settings = $.extend({            //options defaults go here            ...        }, options);        //this applies the plugin to all matching elements        return this.each(function() {            //code of the plugin comes here            ...        });    }})(jQuery);

Мы будем вызывать плагин в файле script.js. В данном уроке мы будем использовать параметры, о которых мы упоминали в предыдущей главе:

  • shadowColor: Цвет тени, которую отбрасывает наш элемент.
  • shadowLength: Длина отбрасываемой тени.
  • shadowAngle: Угол тени.
  • shadowOpacity: Насколько тень размыта или прозрачна.
  • spacing: Об этом параметре мы не упоминали в предыдущей главе. Однако он нам нужен для того, чтобы использовать пространство вокруг элемента, для которого мы создаем длинные тени. Таким образом эффект будет более заметным.

Давайте начнем использование плагина. Для создания длинной тени мы будем использовать HTML5 компонент canvas. Мы можем создать буферный холст canvas, в котором мы нарисуем копию исходного изображения и его тень. Для того, чтобы нарисовать тень, мы будем просто рисовать копии изображения одну над другой с небольшим отступом.

Количество копий и отступ будут высчитываться с помощью простого преобразования полярных координат на основе значений shadowLength и shadowAngle. Также нам нужно будет окрасить эти копии в соответствии с цветом тени, который задан значением shadowColor.

Поскольку мы рисуем тень путем нескольких изображений, расположенных друг над другом, будем рисовать их в обратном порядке, от нижнего к верхнему, начиная с самого дальнего слоя тени. После этого нам необходимо задать прозрачность получившийся тени параметром shadowOpacity.

После того, как мы завершили рисование тени, мы просто нарисуем исходное изображение поверх нее.

Давайте посмотрим, как вышесказанное будет выглядеть в коде файла jquery.longshadows.js:

(function($) {    $.fn.longshadows = function(options) {        var settings = $.extend({            shadowColor: "black",            shadowLength: 100,            shadowAngle: 45,            shadowOpacity: 100,            spacing: 0        }, options);        return this.each(function() {            var img = this;            img.onload = function() {                var self = this;                var canvas = document.createElement("canvas");                var ctx = canvas.getContext("2d");                canvas.width = self.width + settings.spacing;                canvas.height = self.height + settings.spacing;                for (var r = settings.shadowLength; r >= 1; r--) {                    var x = Math.round(r * Math.cos(settings.shadowAngle * Math.PI / 180));                    var y = Math.round(r * Math.sin(settings.shadowAngle * Math.PI / 180));                    ctx.save();                    ctx.translate(x + settings.spacing / 2, y + settings.spacing / 2);                    ctx.drawImage(self, 0, 0);                    ctx.globalCompositeOperation = "source-in";                    ctx.fillStyle = settings.shadowColor;                    ctx.rect(0, 0, canvas.width, canvas.height);                    ctx.fill();                    ctx.restore();                }                                var tempCanvas = copyCanvas(canvas, settings.shadowOpacity / 100.0);                ctx.clearRect(0, 0, canvas.width, canvas.height);                ctx.drawImage(tempCanvas, 0, 0);                ctx.drawImage(self, settings.spacing / 2, settings.spacing / 2);                self.onload = null;                $(self).attr("src", canvas.toDataURL("image/png"));            };            img.src = img.src;        });    };})(jQuery);function copyCanvas(canvas, opacity) {    var canvasDest = document.createElement("canvas");    canvasDest.width = canvas.width;    canvasDest.height = canvas.height;    canvasDest.getContext("2d").globalAlpha = opacity;    canvasDest.getContext("2d").drawImage(canvas, 0, 0);    return canvasDest;}

Конфигурирование плагина может осуществляться путем передачи значений в параметре options. Эти значения будут объединены со значениями по умолчанию и сохраняться в переменной settings. Такой подход дает возможность быстрого использования плагина без необходимости задавать какие-либо параметры.

Переменная img будет содержать ссылку на исходное изображение, для которого мы создаем данный эффект. Мы будем следить за событием onload этого изображения, чтобы применять эффект только после полной его загрузки. Также вы заметите, что после функции onload идет img.src = img.src;. Это позволяет спровоцировать вызов функции onload, поскольку мы не может быть уверенны в том, в каком порядке браузер будет загружать изображение и скрипт.

Функцией onload мы создаем буферный элемент canvas, в котором мы нарисуем конечный результат, элемент будет таких же размеров, как и исходное изображение, плюс spacing. После этого, начиная с наиболее удаленной от центра точки, мы нарисуем копии изображения в элементе canvas, используя преобразование полярных координат на значение отступа для изображения:

var x = Math.round(r * Math.cos(settings.shadowAngle * Math.PI / 180));var y = Math.round(r * Math.sin(settings.shadowAngle * Math.PI / 180));

Для рисования изображения на холсте canvas, мы будем использовать 2D контекст холста и вызывать функцию drawImage(). Так будет нарисована копия изображения на холсте, но нам нужна ее окрашенная версия. Для этого мы будем использовать операции композитинга canvas. В нашем случае мы используем source-in вместе с прямоугольником, окрашенным цветом shadowColor, что приведет к тому, что будет нарисовано изображение такой же формы, как и исходное, но окрашенное в цвет shadowColor.

Обратите внимание, что если ваше изображение имеет много цветов, результат будет только одного цвета, заданного shadowColor, что является желаемым результатом в нашем случае, поскольку мы рисуем тень, которая обычно одноцветная.

Цикл for нужен для рисования тени, однако в нем тень рисуется полностью непрозрачной. Мы бы хотели иметь возможность задавать прозрачность тени параметром shadowOpacity. Для этого мы используем функцию copyCanvas(), которая использует временный холст и устанавливает заданное значение прозрачности для содержимого холста.

Выполнять данную операцию нам нужно в самом конце, когда вся тень уже была нарисована, иначе наложение прозрачных элементов друг на друга приведет к созданию эффекта градиента.

Давайте посмотрим на две последние строчки функции onload:

self.onload = null;$(self).attr("src", canvas.toDataURL("image/png"));

В первой строчке мы удаляем функцию onload для исходного изображения. Это сделано, потому что в следующей строке мы хотим использовать нарисованное изображение в качестве нового значения src исходного изображения. Если бы мы не обнулили эту функцию, это бы привело к зацикливанию.

Как пользоваться jQuery плагином для создания длинных теней?

После того, как мы написали плагин, давайте посмотрим, как мы может его использовать, и к каким результатом это приведет. Для этого мы возьмем файл script.js, в котором мы будет вызывать только что созданный плагин.

Самый просто способ вызвать плагин это:

$(document).ready(function(){    $("#logo").longshadows();});

Этот код дает браузеру команду, чтобы по завершении загрузки страницы плагин longshadows был вызван для элемента со значением аттрибута ID logo.

Подобный вызов плагина приведет к использованию значений его параметров по умолчанию. Поскольку результат нас не очень устраивает при таких значениях, давайте посмотрим, что мы можем изменить. Давайте вызовем плагин вот таким образом:

$(document).ready(function(){    $("#logo").longshadows({        spacing:50,        shadowOpacity:30,        shadowAngle:30,        shadowLength:400    });});

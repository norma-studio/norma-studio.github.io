Прежде чем приступать к данному уроку, было бы неплохо обобщить все, чем мы занимались до этого, чтобы исключить непонимание и ошибки. Если вы пропустили какой-либо из шагов предыдущих трех уроков, советую вернуться и внести необходимые изменения.

Во втором уроке мы создали три разных файла: country.ts, country-data.ts и country.service.ts. Файл country.ts используется для хранения определения класса Country для того, чтобы мы могли импортировать его в других файлах. Файл country-data.ts используется для хранения массива COUNTRIES.

Этот массив содержит объекты всех стран, которые мы хотим отображать в нашем приложении. Файл country.service.ts используется для определения класса CountryService со всеми методами, которые мы будем использовать для доступа к информации о различных странах. Методы класса CountryService используются для вывода стран с наибольшими показателями по таким критериям как население и площадь, или для поиска информации о стране по названию этой страны.

В третьем уроке мы создали HomeComponent для нашего приложения. Мы это сделали путем создания трех разных файлов: home.component.ts, home.component.html и home.component.css. Файл home.component.ts содержал определение класса HomeComponent с различными методами и свойствами для доступа и хранения информации обо всех странах.

Методы в классе HomeComponent основывались на методах, определенных в country.service.ts для получения всех данных. Файл home.component.html используется для шаблона, который будет использоваться для отображения всех данных, полученных с помощью методов в файле home.component.ts. Файл home.component.css используется для создания различных стилевых правил, которые будут определять внешний вид различных элементов в шаблоне.

В четвертом уроке мы создали еще два компонента для нашего приложения. Для компонента AllCountries мы создали файлы: all-countries.component.ts, all-countries.component.html и all-countries.component.css. Для компонента CountryDetail мы создали файлы: country-detail.component.ts, country-detail.component.html и country-detail.component.css. Так же как и для HomeComponent, файлы .ts содержат функционал компонентов, файлы .html содержат шаблон, и фалы .css содержат различные правила, применяемые для элементов в файле шаблона.

В этом уроке мы будем добавлять маршрутизацию в наше приложение. Таким способом мы сможем с легкостью перемещаться между различными компонентами.

Изменения оболочки приложения

Теперь нам необходимо внести изменения в оболочку приложения, чтобы наше приложение начало работать. Файл app.component.ts будет выглядеть точно так же, как и в первом уроке.

import { Component } from "@angular/core";@Component({  selector: "app-root",  templateUrl: "./app.component.html",  styleUrls: ["./app.component.css"]})export class AppComponent {  title = "Fun Facts About Countries";}

Однако, мы внесем изменения в файл app.component.html. Файл HTML теперь должен содержать следующий код:

<h1>{{title}}</h1><nav>  <a routerLink="/home">Go Back to Homepage</a>  <a routerLink="/all-countries">List of all Countries</a></nav><router-outlet></router-outlet>

Ранее мы отображали только заголовок приложения. Теперь мы также добавили в шаблон и навигацию. Директива routerLink используется для получения ссылок к различным областям или страницам вашего приложения. Angular определяет компонент, который необходимо отобразить, при помощи директивы routerLink. Позиция, в которую эти компоненты извлекаются задается routerOutlets. Компоненты извлекаются после тегов router-outlet.

После создания файла шаблона добавим следующий код CSS в app.component.css для задания стиля ссылкам навигации и заголовку:

nav {  margin: 0px;  text-align: center;}h1 {  font-family: "Lato";  color: #999;  text-align: center;}h2 {  font-size: 2em;  margin-top: 0;  padding-top: 0;}nav a {  padding: 10px;  text-decoration: none;  margin: 10px 0px;  display: inline-block;  background-color: black;  color: white;  border-radius: 5px;  font-family: "Lato";}nav a:hover {  background-color: gray;}nav a.active {  color: #039be5;}

Теперь мы будем сообщать Angular, какие компоненты необходимо извлекать для определенного маршрута или адреса. Создайте файл app-routing.module.ts в директории src/app и добавьте в него следующий код:

import { NgModule } from "@angular/core";import { RouterModule, Routes } from "@angular/router";import { HomeComponent } from "./home/home.component";import { AllCountriesComponent } from "./all-countries/all-countries.component";import { CountryDetailComponent } from "./country-detail/country-detail.component";const routes: Routes = [    { path: "", redirectTo: "/home", pathMatch: "full" },    { path: "home", component: HomeComponent },    { path: "detail/:name", component: CountryDetailComponent },    { path: "all-countries", component: AllCountriesComponent }];@NgModule({    imports: [RouterModule.forRoot(routes)],    exports: [RouterModule]})export class AppRoutingModule { }

Начнем с того, что импортируем все необходимые зависимости, включая компоненты, которые мы хотим извлекать для различных маршрутов. После этого мы определяем различные адреса и компоненты, которые должны извлекаться, когда пользователь посещает эти адреса. Вы также можете использовать переадресацию, что мы делали для приложения информации о стране ранее. Когда пользователи посещают http://localhost:4200/, они будут переадресованы на http://localhost:4200/home. Обратите внимание, что указание переадресации требует, чтобы вы указали значение свойства pathMatch, которое сообщает обработчику, как он должен сопоставлять URL c адресом того или иного маршрута.

Когда пользователь посещает http://localhost:4200/all-countries, мы будем извлекать AllCountriesComponent после тега router-outlet в файле app.component.html для отображения списка всех стран.

Мы использовали директиву routerLink в шаблоне AllCountriesComponent, а также в HomeComponent для вывода ссылки, которую пользователи могут кликнуть для того, чтобы получить подробности о любой стране. Значение routerLink для каждой страны, которая выводиться в этих шаблонах, должно соответствовать формату routerLink="/detail/{{country.name}}". Значение свойства path для вывода CountryDetailComponent было определено как detail/:name, для соответствия значению директивы routerLink. Часть :name в этом пути используется для определения названия страны.

Обновление файла app.module.ts

Последнее, что нам нужно сделать для того, чтобы получить полностью рабочее приложение Angular, это обновить файл app.module.ts. Если вы откроете этот файл в текстовом редакторе, то заметите, что все три компонента, которые мы сгенерировали, используя Angular CLI, уже были импортированы в этот файл. До внесения каких-либо изменений, ваш файл app.module.ts должен содержать следующий код:

import { BrowserModule } from "@angular/platform-browser";import { NgModule } from "@angular/core";import { AppComponent } from "./app.component";import { CountryService } from "./country.service";import { HomeComponent } from "./home/home.component";import { AllCountriesComponent } from "./all-countries/all-countries.component";import { CountryDetailComponent } from "./country-detail/country-detail.component";@NgModule({  declarations: [    AppComponent,    HomeComponent,    AllCountriesComponent,    CountryDetailComponent  ],  imports: [    BrowserModule  ],  providers: [CountryService],  bootstrap: [AppComponent]})export class AppModule { }

В файле app.module.ts нам нужно сделать только два изменения. Во-первых, на нужно импортировать класс AppRoutingModule из файла app-routing.module.ts, который мы создали в предыдущем разделе урока. Во-вторых, добавить этот класс в массив @NgModule.providers. После этих изменений, ваш файл app.module.ts должен выглядеть вот так:

import { BrowserModule } from "@angular/platform-browser";import { NgModule } from "@angular/core";import { AppComponent } from "./app.component";import { CountryService } from "./country.service";import { HomeComponent } from "./home/home.component";import { AllCountriesComponent } from "./all-countries/all-countries.component";import { CountryDetailComponent } from "./country-detail/country-detail.component";import { AppRoutingModule } from "./app-routing.module";@NgModule({  declarations: [    AppComponent,    HomeComponent,    AllCountriesComponent,    CountryDetailComponent  ],  imports: [    BrowserModule,    AppRoutingModule  ],  providers: [CountryService],  bootstrap: [AppComponent]})export class AppModule { }

Если вы перейдете в каталог вашего приложения и введете в консоли следующую команду, ваше приложение загрузит и выведет HomeComponent.

ng serve --open

Вы можете кликнуть на блоках различных стран или на ссылках в навигации для загрузки различных компонентов.

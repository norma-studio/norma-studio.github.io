Создание новых режимов смешивания

По умолчанию CamanJS предлагает десять режимов смешивания. Это normal, multiply, screen, overlay, difference, addition, exclusion, softLight, lighten, и darken. Библиотека также позволяет вам регистрировать свои собственные режимы наложения. Таким образом, вы можете контролировать, как соответствующие пиксели текущего слоя и родительского слоя смешиваются вместе для получения конечного результата.

Вы можете создать новый режим смешивания, используя Caman.Blender.register("blend_mode", callback);. Здесь blend_mode - это имя, которое вы хотите использовать, чтобы идентифицировать режим наложения, который вы создаете. Функция обратного вызова принимает два параметра, которые содержат значения RGB для разных пикселей на текущем уровне и соответствующие пиксели на родительском уровне. Функция возвращает объект с конечными значениями для каналов rgb.

Ниже приведен пример настраиваемого режима смешивания, который устанавливает значение отдельных каналов пикселя в 255, если значение этого канала для соответствующего пикселя в родительском слое превышает 128. Если значение ниже 128, конечное значение канала является результатом вычитания значения канала текущего слоя из значения родительского канала. Название этого режима наложения - maxrgb.

Caman.Blender.register("maxrgb", function(rgbaLayer, rgbaParent) {    return {        r: rgbaParent.r > 128 ? 255 : rgbaParent.r - rgbaLayer.r,        g: rgbaParent.g > 128 ? 255 : rgbaParent.g - rgbaLayer.g,        b: rgbaParent.b > 128 ? 255: rgbaParent.b - rgbaLayer.b    };});

Давайте создадим другой режим смешивания аналогичным образом. На этот раз конечные значения канала будут установлены на 0, если значение канала для соответствующего пикселя в родительском слое больше 128. Если значение канала для родительского уровня меньше 128, конечным результатом будет добавление значений канала для текущего слоя и родительского уровня конкретного пикселя. Этот режим наложения был назван minrgb.

Caman.Blender.register("minrgb", function(rgbaLayer, rgbaParent) {    return {        r: rgbaParent.r < 128 ? rgbaParent.r + rgbaLayer.r : 0,        g: rgbaParent.g < 128 ? rgbaParent.g + rgbaLayer.r : 0,        b: rgbaParent.b < 128 ? rgbaParent.r + rgbaLayer.r : 0    };});

Вам стоит попробовать создать свои собственные режимы наложения для практики.

Создание новых пиксельных фильтров

В CamanJS есть две широкие категории фильтров. Вы можете либо работать со всем изображением по одному пикселю за раз, либо изменять изображение с помощью ядра свертки. Ядро свертки - это матрица, которая определяет цвет определенного пикселя на основе пикселей вокруг него. В этом разделе мы сосредоточимся на фильтрах на основе пикселей. Манипуляции ядра будут рассмотрены в следующем разделе.

Фильтрам на основе пикселей присваивается значение RGB-каналов для одного пикселя за раз. Конечные значения RGB для этого конкретного пикселя не зависят от окружающих пикселей. Вы можете создавать свои собственные фильтры, используя Caman.Filter.register("filter_name", callback);. Любой создаваемый фильтр должен вызывать метод process(). Этот метод принимает имя фильтра и функцию обратного вызова в качестве параметров.

В следующем фрагменте кода показано, как создать фильтр на основе пикселей, который превращает изображения в оттенки серого. Это делается путем вычисления люминесценции каждого пикселя, а затем установки значения отдельных каналов равным рассчитанной люминесценции.

Caman.Filter.register("grayscale", function () {    this.process("grayscale", function (rgba) {        var lumin = (0.2126 * rgba.r) + (0.7152 * rgba.g) + (0.0722 * rgba.b);        rgba.r = lumin;        rgba.g = lumin;        rgba.b = lumin;    });    return this;});

Вы можете создать пороговый фильтр аналогичным образом. На этот раз мы разрешим пользователям передавать пороговое значение. Если яркость определенного пикселя выше установленного пользователем предела, этот пиксель станет белым. Если яркость определенного пикселя меньше предела, указанного пользователем, этот пиксель станет черным.

Caman.Filter.register("threshold", function (limit) {    this.process("threshold", function (rgba) {        var lumin = (0.2126 * rgba.r) + (0.7152 * rgba.g) + (0.0722 * rgba.b);        rgba.r = lumin > limit ? 255 : 0;        rgba.g = lumin > limit ? 255 : 0;        rgba.b = lumin > limit ? 255 : 0;    });    return this;});

В качестве упражнения вы должны попытаться создать свои собственные фильтры на основе пикселей, которые, например, увеличивают значение для определенного канала на всех пикселях.

Вместо того, чтобы манипулировать цветом текущего пикселя, CamanJS также позволяет вам устанавливать цвет для пикселей в абсолютных и относительных местоположениях. К сожалению, это поведение немного ошибочно, поэтому нам придется переписать некоторые методы. Если вы посмотрите на исходный код библиотеки, вы заметите, что такие методы, как getPixel() и putPixel(), вызывают методы coordinatesToLocation() и locationToCoordinates() на this. Однако, эти методы определены не на прототипе, а на самом классе.

Другая проблема с библиотекой заключается в том, что метод putPixelRelative() использует имя переменной nowLoc вместо newLoc в двух разных местах. Вы можете избавиться от этих двух проблем, добавив в свой скрипт следующий код.

Caman.Pixel.prototype.coordinatesToLocation = Caman.Pixel.coordinatesToLocationCaman.Pixel.prototype.locationToCoordinates = Caman.Pixel.locationToCoordinatesCaman.Pixel.prototype.putPixelRelative = function (horiz, vert, rgba) {    var newLoc;    if (this.c == null) {        throw "Requires a CamanJS context";    }    newLoc = this.loc + (this.c.dimensions.width * 4 * (vert * -1)) + (4 * horiz);    if (newLoc > this.c.pixelData.length || newLoc < 0) {        return;    }    this.c.pixelData[newLoc] = rgba.r;    this.c.pixelData[newLoc + 1] = rgba.g;    this.c.pixelData[newLoc + 2] = rgba.b;    this.c.pixelData[newLoc + 3] = rgba.a;    return true;};

После исправления кода вы должны создать фильтр, который основан на putPixelRelative() без каких-либо проблем. Вот один такой фильтр, который я создал.

Caman.Filter.register("erased", function (adjust) {    this.process("erased", function (rgba) {        if(Math.random() < 0.25) {        rgba.putPixelRelative(2, 2, {            r: 255,            g: 255,            b: 255,            a: 255        });        }    });    return this;});

Этот фильтр случайным образом устанавливает значение пикселей в две строки вверх и два столбца справа от текущего пикселя до белого. Это стирает части изображения. Отсюда и название фильтра.

Создание новых фильтров на основе манипуляции с ядром

Как я упоминал ранее, CamanJS позволяет создавать собственные фильтры, где цвет текущего пикселя определяется пикселями, окружающими его. В основном, эти фильтры передают каждый пиксель в изображении, которое вы редактируете. Пиксель в изображении будет окружен восемью другими пикселями. Значения этих девяти пикселей из изображения умножаются на соответствующие записи матрицы свертки. Все эти результаты затем добавляются вместе, чтобы получить окончательное значение цвета для пикселя. Подробнее о процессе можно прочитать в документации GIMP.

Так же, как и пиксельные фильтры, вы можете определить свои собственные фильтры манипуляции ядром с помощью Caman.Filter.register("filter_name", callback);. Единственное отличие состоит в том, что теперь вы будете вызывать process Kernel() внутри функции обратного вызова.

Вот пример создания фильтра emboss с помощью манипуляции с ядром.

Caman.Filter.register("emboss", function () {    this.processKernel("emboss", [        -2, -1, 0,        -1, 1, 1,        0, 1, 2    ]);});

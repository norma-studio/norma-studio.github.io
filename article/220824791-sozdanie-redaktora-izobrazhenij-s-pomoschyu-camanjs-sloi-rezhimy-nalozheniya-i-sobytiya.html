В предыдущем учебном пособии вы узнали, как создать редактор изображений с помощью CamanJS, который может применять к изображению базовые фильтры, такие как контраст, яркость и шум. CamanJS также имеет некоторые другие встроенные фильтры, такие как Nostalgia, Pinhole, Sunrise и т. д., которые мы применили непосредственно к изображению.

В этом уроке мы рассмотрим некоторые дополнительные функции библиотеки, такие как слои, режимы наложения и события.

Слои в CamanJS

В первом уроке мы работали только с одним слоем, содержащим наш образ. Все фильтры, которые мы применяли, обрабатывали только этот основной слой. CamanJS позволяет создавать несколько слоев для более сложного редактирования изображений. Вы можете создавать вложенные слои, но они всегда будут применяться к родительскому слою как стек.

Всякий раз, когда вы создаете новый слой и применяете его к родительскому слою, по умолчанию используется режим наложения normal. Вы можете создать новый слой на холсте, используя метод newLayer(). При создании нового слоя можно также передать функцию обратного вызова, которая будет полезна при работе со слоем.

Эта функция может быть использована для многих задач, таких как настройка режима наложения для нового слоя с помощью метода setBlendingMode(). Аналогичным образом можно управлять непрозрачностью нового слоя с помощью метода opacity().

Любой новый слой, который вы создаете, может быть залит сплошным цветом с помощью метода fillColor(). Можно также скопировать содержимое родительского слоя в новый слой с помощью метода copyParent(). Все фильтры, о которых мы узнали в предыдущем уроке, также могут быть применены к новому слою, который мы создаем. Например, можно увеличить яркость вновь созданного слоя с помощью this.filter.brightness(10).

Вместо копирования родительского изображения или заливки слоя сплошным цветом можно также загрузить любое другое изображение в слой и наложить его на родительский. Так же, как основное изображение, вы сможете применять различные фильтры к наложенному изображению.

В следующем фрагменте кода мы прикрепили обработчик событий click к трем кнопкам, которые заполнят новый слой сплошным цветом, родительский слой и наложенное изображение соответственно. 

$("#orange-btn").on("click", function (e) {    Caman("#canvas", function () {        this.newLayer(function () {            this.opacity(50);            this.fillColor("#ff9900");        });        this.render();    });});$("#parent-btn").on("click", function (e) {    Caman("#canvas", function () {        this.newLayer(function () {            this.opacity(50);            this.copyParent();            this.filter.brightness(20);        });        this.render();    });});$("#overlay-btn").on("click", function (e) {    var oImg = new Image();    oImg.src = "trees.png";        Caman("#canvas", function () {        this.newLayer(function () {            this.opacity(50);            this.overlayImage(oImg);            this.filter.brightness(20);        });        this.render();    });});

Режимы наложения в CamanJS

В предыдущем разделе мы сохраняли непрозрачность любого нового слоя, добавленного на холст, ниже 100. Это было сделано потому, что в противном случае новый слой полностью скроет старый. Когда вы размещаете один слой поверх другого, CamanJS позволяет указать режим наложения, который определяет конечный результат после размещения. Режим наложения по умолчанию установлен в normal.

Это означает, что любой новый слой, добавленный на холст, сделает слой под ним невидимым. Всего в библиотеке десять режимов наложения. Это normal, multiply, screen, overlay, difference, addition, exclusion, softLight, exclusion, и darken.

Как я уже упоминал ранее, режим наложения normal устанавливает окончательный цвет равным цвету нового слоя. Режим наложения multiply определяет конечный цвет пикселя путем умножения отдельных каналов и деления результата на 255. Режимы смешения difference и addition работают аналогичным образом, но они вычитают и добавляют каналы.

Режим наложения darken устанавливает конечный цвет пикселя равным наименьшему значению отдельных цветовых каналов. Режим наложения lighten устанавливает конечный цвет пикселя равным наибольшему значению отдельных цветовых каналов. Режим наложения exclusion чем-то похож на difference, но он устанавливает контраст на более низкое значение. В случае режима наложения screen окончательный цвет получается инвертированием цветов каждого слоя, умножением их, а затем снова инвертированием результата.

Режим наложения overlay действует как multiply, если нижний цвет темнее, и он действует как screen, если нижний цвет светлее.

Если вы хотите, чтобы цвета в разных слоях взаимодействовали по-разному, CamanJS также позволяет определять собственные режимы наложения. Мы рассмотрим это в следующем уроке курса.

Ниже приведен код JavaScript для применения различных режимов наложения к изображению:

$("#multiply-btn").on("click", function (e) {    hexColor = $("#hex-color").val();    Caman("#canvas", function () {        this.revert(false);        this.newLayer(function () {            this.fillColor(hexColor);            this.setBlendingMode("multiply");        });        this.render();    });});$("#screen-btn").on("click", function (e) {    hexColor = $("#hex-color").val();    Caman("#canvas", function () {        this.revert(false);        this.newLayer(function () {            this.fillColor(hexColor);            this.setBlendingMode("screen");        });        this.render();    });});

В приведенном выше фрагменте кода, мы получаем шестнадцатеричное значение цвета в поле ввода. Затем этот цвет применяется к новому слою. Аналогичным образом можно написать код для применения других режимов наложения.

Попробуйте указать выбранный цвет в поле ввода, а затем применить любой из режимов наложения, нажав на соответствующую кнопку. В этом примере я применил режимы наложения к сплошному цвету, но вы также можете применить их к наложенному изображению из предыдущего раздела.

События в CamanJS

Если вы загрузили любое большое изображение в демо-версию первого или второго учебника, вы могли заметить, что результат любого примененного фильтра или режима наложения появлялся довольно долго.

Большие изображения имеют много пикселей, и вычисление конечного значения различных каналов для каждого пикселя после применения определенного режима наложения может занять очень много времени. Например, при применении режима наложения multiply на изображение с размерами 1920*1080 устройство должно будет выполнить умножение и деление более 6 миллионов раз.

В таких случаях можно использовать события, которые дадут пользователям некоторое представление о ходе выполнения фильтра или режима наложения. CamanJS имеет пять различных событий, которые могут быть использованы для выполнения конкретных функций обратного вызова на разных этапах. Вот эти пять событий processStart, processComplete, renderFinished, blockStarted, и blockFinished.

События processStart и processComplete запускаются после запуска или завершения процесса отрисовки одним фильтром. После применения всех указанных фильтров к изображению библиотека запускает событие renderFinished.

CamanJS делит большие изображения на блоки, прежде чем начать манипулировать ими. События blockStarted и blockFinished запускаются после обработки библиотекой отдельных блоков изображения.

В нашем примере мы будем использовать только события processStart и renderFinished для информирования пользователей о ходе выполнения операции редактирования изображений.

Caman.Event.listen("processStart", function (process) {    $(".process-message").text("Applying " + process.name);});Caman.Event.listen("renderFinished", function () {    $(".process-message").text("Done!");});

С событиями processStart и processFinish вы получаете доступ к имени процесса, в настоящее время работающего над изображением. События blockStarted и blockFinished, с другой стороны, дают вам доступ к такой информации, как общее количество блоков, текущий обрабатываемый блок и количество готовых блоков.

Попробуйте нажать на любую кнопку в демо ниже, и вы увидите название процесса, в настоящее время манипулирующего изображением в области под холстом.

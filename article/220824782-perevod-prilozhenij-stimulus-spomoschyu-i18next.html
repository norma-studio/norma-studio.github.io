В моей предыдущей статье я рассмотрел Stimulus - скромнуюструктуру JavaScript, созданную Basecamp. Сегодня я расскажу оинтернационализации приложения Stimulus, поскольку структура не предоставляетникаких инструментов I18n из коробки. Интернационализация - важный шаг, особенно когда вашеприложение используется людьми со всего мира, поэтому базовое понимание того,как это сделать, действительно может пригодиться.

Разумеется, вам решать, какое решение дляинтернационализации реализовать, будь то jQuery.I18n, Polyglot или какой-либодругой. В этом уроке я хотел бы показать вам популярную инфраструктуру I18n подназванием I18next, которая имеет множество интересных функций и предоставляетмножество дополнительных сторонних плагинов, чтобы еще больше упростить процессразработки. Даже со всеми этими функциями I18next не является сложныминструментом, и вам не нужно изучать много документации для начала работы.

В этой статье вы узнаете, как включить поддержку I18n вприложениях Stimulus с помощью библиотеки I18next. В частности, мы поговорим о:

  • Конфигурация I18next
  • файлы переводов и их асинхронная загрузка
  • выполнение переводов и перевод всей страницы за один раз
  • работа с множественными числами и гендерной информацией
  • переключение между локалями и сохранение выбранной локали в параметре GET
  • настройка языкового стандарта на основе предпочтенийпользователя

Исходный код доступен в учебнике GitHub repo.

Загрузите приложение Stimulus

Чтобы начать работу, давайте клонируем проект StimulusStarter и установите все зависимости с помощью менеджера пакетов Yarn:

git clone https://github.com/stimulusjs/stimulus-starter.gitcd stimulus-starteryarn install

Мы собираемся создать простое веб-приложение, котороезагружает информацию о зарегистрированных пользователях. Для каждогопользователя мы покажем его / ее логин и количество фотографий, которые онзагрузил до сих пор (на самом деле не имеет значения, что это за фото).

Кроме того, мы представим языковой коммутатор в верхнейчасти страницы. Когда язык выбран, интерфейс должен быть переведен сразу безперезагрузки страницы. Кроме того, URL-адрес должен быть добавлен в ?locale GET, указывающий, какой язык в настоящее время используется.Конечно, если страница загружена с указанным параметром, правильный язык долженбыть установлен автоматически.

Хорошо, давайте продолжим оказание нашим пользователям.Добавьте следующую строку кода в файл public / index.html:

<div data-controller="users" data-users-url="/api/users/index.json"></div>

Здесь мы используем пользовательский контроллер ипредоставляем URL-адрес, с которого можно загрузить наших пользователей. Вреальном приложении у нас, вероятно, есть серверный скрипт, который извлекаетпользователей из базы данных и отвечает на JSON. Однако для этого урока просто поместим все необходимыеданные в файл public/api/users/index.json:

[  {    "login": "johndoe",    "photos_count": "15",    "gender": "male"  },  {    "login": "annsmith",    "photos_count": "20",    "gender": "female"  }]

Теперь создайте новый файлsrc/controller/users_controller.js:

import { Controller } from "stimulus"export default class extends Controller {  connect() {    this.loadUsers()  }}

Как только контроллер подключается к DOM, мы асинхронно загружаем нашихпользователей с помощью метода loadUsers():

  loadUsers() {    fetch(this.data.get("url"))    .then(response => response.text())    .then(json => {      this.renderUsers(json)    })  }

Этот метод отправляет запрос на выборку к данномуURL-адресу, захватывает ответ и, наконец, предоставляет пользователям:

  renderUsers(users) {    let content = ""    JSON.parse(users).forEach((user) => {      content += `<div>Login: ${user.login}<br>Has uploaded ${user.photos_count} photo(s)</div><hr>`    })    this.element.innerHTML = content  }

renderUsers(), в свою очередь, анализирует JSON, строитновую строку со всем содержимым и, наконец, отображает это содержимое настранице (this.element будет возвращать фактический узел DOM, к которомуподключен контроллер, который является div в нашем случае).

I18next

Теперь мы перейдем к интеграции I18next в наше приложение.Добавьте в наш проект две библиотеки: I18next и плагин для асинхронной загрузкифайлов перевода с обратной стороны:

yarn add i18next i18next-xhr-backend

Мы собираемся хранить все связанные с I18next вещи вотдельном файле src / i18n / config.js, поэтому создадим его сейчас:

import i18next from "i18next"import I18nXHR from "i18next-xhr-backend"const i18n = i18next.use(I18nXHR).init({  fallbackLng: "en",  whitelist: ["en", "ru"],  preload: ["en", "ru"],  ns: "users",  defaultNS: "users",  fallbackNS: false,  debug: true,  backend: {    loadPath: "/i18n/{{lng}}/{{ns}}.json",  }}, function(err, t) {  if (err) return console.error(err)}); export { i18n as i18n }

Давайте пойдем сверху вниз, чтобы понять, что здесьпроисходит:

  • use(I18nXHR) включает плагин i18next-xhr-backend.
  • fallbackLngговорит, что он использует английский как запасной язык.
  • whitelist позволяет установить только английский и русскийязыки. Конечно, вы можете выбрать любые другие языки.
  • preload указывает, что файлы перевода должны бытьпредварительно загружены с сервера, а не загружать их при выборесоответствующего языка.
  • ns означает «пространство имен» и принимает либо строку,либо таблицу. В этом примере у нас есть только одно пространство имен, но дляболее крупных приложений вы можете ввести другие пространства имен, такие какadmin, cart, profile и т. д. Для каждого пространства имен необходимо создать отдельныйфайл перевода.
  • defaultNS устанавливает для пользователей пространство именпо умолчанию.
  • fallbackNS отключает резервное копирование пространстваимен.
  • deubg позволяет отображать отладочную информацию в консолибраузера. В частности, в нем указано, какие файлы переводов загружены, какойязык выбран и т. д. Вероятно, вы захотите отключить этот параметр передразвертыванием приложения для производства.
  • backend предоставляет конфигурацию для плагина I18nXHR иопределяет, где загружать переводы. Обратите внимание, что путь долженсодержать заголовок локали, тогда как файл должен быть назван в честьпространства имен и иметь расширение .json
  • function(err, t) - это обратный вызов для запуска, когдаI18next готов (или когда была обнаружена ошибка).

Затем давайте создадим файлы переводов. Переводы русскогоязыка должны быть помещены в файл public / i18n / ru / users.json:

{  "login": "Логин"}

login здесь - это ключ перевода, тогда как Логин - этозначение для отображения.

Английские переводы, в свою очередь, должны перейти к файлуpublic / i18n / en / users.json:

{  "login": "Login"}

Чтобы убедиться, что I18next работает, вы можете добавитьследующую строку кода в обратный вызов внутри файла i18n / config.js:

// config goes here...function(err, t) {  if (err) return console.error(err)  console.log(i18n.t("login"))}

Здесь мы используем метод t, означающий «translate». Этотметод принимает ключ перевода и возвращает соответствующее значение.

Тем не менее, у нас может быть много частейпользовательского интерфейса, которые необходимо перевести, и это, используяметод t, будет довольно утомительным. Вместо этого я предлагаю вам использоватьдругой плагин, называемый loc-i18next, который позволяет вам переводить сразунесколько элементов.

Перевод в One Go

Установите плагин loc-i18next:

yarn add loc-i18next

Импортируйте его вверху файла src / i18n / config.js:

import locI18next from "loc-i18next"

Теперь предоставьте конфигурацию самого модуля:

// other configconst loci18n = locI18next.init(i18n, {  selectorAttr: "data-i18n",  optionsAttr: "data-i18n-options",  useOptionsAttr: true});export { loci18n as loci18n, i18n as i18n }

Здесь есть несколько вещей:

  • locI18next.init(i18n) создает новый экземпляр плагина наоснове ранее определенного экземпляра I18next.
  • selectorAttr указывает, какой атрибут использовать дляобнаружения элементов, требующих локализации. В принципе, loc-i18next собираетсяискать такие элементы и использовать значение атрибута data-i18n в качествеключа перевода.
  • optionsAttr указывает, какой атрибут содержит дополнительныепараметры перевода.
  • useOptionsAttr указывает плагину использовать дополнительныепараметры.

Наши пользователи загружаются асинхронно, поэтому нам нужноподождать, пока эта операция не будет выполнена, и после этого выполнитьлокализацию. На данный момент давайте просто установим таймер, который долженподождать две секунды, прежде чем вызывать метод localize() - это временныйвзлом, конечно.

  import { loci18n } from "../i18n/config"    // other code...    loadUsers() {    fetch(this.data.get("url"))    .then(response => response.text())    .then(json => {      this.renderUsers(json)      setTimeout(() => { // <---        this.localize()      }, "2000")    })  }

Напишите код для метода localize():

  localize() {    loci18n(".users")  }

Как вы видите, нам нужно только передать селектор плагинуloc-i18next. Все элементы внутри (которые имеют набор атрибутов data-i18n)будут автоматически локализованы.

Теперь настройте метод renderUsers. Пока давайте переведемтолько слово «Login»:

  renderUsers(users) {    let content = ""    JSON.parse(users).forEach((user) => {      content += `<div class="users">ID: ${user.id}<br><span data-i18n="login"></span>: ${user.login}<br>Has uploaded ${user.photos_count} photo(s)</div><hr>`    })    this.element.innerHTML = content  }

Замечательно! Перезагрузите страницу, подождите две секундыи убедитесь, что слово «Логин» появляется для каждого пользователя.

Количество и пол

У нас есть локализованная часть интерфейса, чтодействительно круто. Тем не менее, у каждого пользователя есть еще два поля:количество загруженных фотографий и пол. Поскольку мы не можем предсказать, сколько фотографий укаждого пользователя будет, слово «фото» должно быть образовано должным образомна основе данного подсчета. Для этого нам понадобится атрибут data-i18n-options,настроенный ранее. Чтобы предоставить счет, data-i18n-options должны бытьназначены следующим объектом: {"count": YOUR_COUNT}.

Следует также учитывать гендерную информацию. Слово«загружено» на английском языке может быть применено как к мужчинам, так и кженщинам, но по-русски оно становится либо «загрузил», либо «загрузила»,поэтому нам нужны data-i18n-options снова, что имеет { "context": "GENDER" } как значение. Обратите внимание, кстати, что вы можете использовать этотконтекст для достижения других задач, а не только для предоставления гендернойинформации.

  renderUsers(users) {    let content = ""    JSON.parse(users).forEach((user) => {      content += `<div class="users"><span data-i18n="login"></span>: ${user.login}<br><span data-i18n="uploaded" data-i18n-options="{ "context": "${user.gender}" }"></span> <span data-i18n="photos" data-i18n-options="{ "count": ${user.photos_count} }"></span></div><hr>`    })    this.element.innerHTML = content  }

Теперь обновите английские переводы:

{  "login": "Login",  "uploaded": "Has uploaded",  "photos": "one photo",  "photos_plural": "{{count}} photos"}

Здесь нет ничего сложного. Поскольку для английского языка мы незаботимся о гендерной информации (которая является контекстом), ключ переводадолжен быть просто uploaded. Чтобы обеспечить правильные плюрализированные переводы, мыиспользуем photos и photos_plural ключи. Часть {{count}} являетсяинтерполяцией и будет заменена фактическим числом.

Что касается русского языка, все сложнее:

{  "login": "Логин",  "uploaded_male": "Загрузил уже",  "uploaded_female": "Загрузила уже",  "photos_0": "одну фотографию",  "photos_1": "{{count}} фотографии",  "photos_2": "{{count}} фотографий"}

Прежде всего, обратите внимание, что у нас есть оба файлаuploaded_male и uploaded_female для двух возможных контекстов. Далее, правила плюрализациитакже более сложны на русском языке, чем на английском, поэтому мы должныпредоставить не две, а три возможные фразы. I18next поддерживает многие языки из коробки, и этотнебольшой инструмент поможет вам понять, какие ключи плюрализации должны бытьуказаны для заданного языка.

Переключение локалей

Мы выполнили перевод нашего приложения, но пользователидолжны иметь возможность переключаться между локалями. Поэтому добавьте новыйкомпонент «language switcher» в файл public / index.html:

<ul data-controller="languages" class="language-switcher"></ul>

Создайте соответствующий контроллер внутри файла src /controller / languages_controller.js:

import { Controller } from "stimulus"import { i18n, loci18n } from "../i18n/config"export default class extends Controller {    initialize() {      let languages = [        {title: "English", code: "en"},        {title: "Русский", code: "ru"}      ]      this.element.innerHTML = languages.map((lang) => {        return `<li data-action="click->languages#switchLanguage"        data-lang="${lang.code}">${lang.title}</li>`      }).join("")    }}

Здесь мы используем обратный вызов initialize() дляотображения списка поддерживаемых языков. Каждый li имеет атрибутaction-action, который указывает, какой метод (switchLanguage, в данном случае)должен быть запущен при нажатии элемента.

Теперь добавьте метод switchLanguage():

  switchLanguage(e) {    this.currentLang = e.target.getAttribute("data-lang")  }

Он просто использует цель события и захватывает значениеатрибута data-lang.

Я также хотел бы добавить getter и setter для атрибутаcurrentLang:

  get currentLang() {    return this.data.get("currentLang")  }  set currentLang(lang) {    if(i18n.language !== lang) {      i18n.changeLanguage(lang)    }    if(this.currentLang !== lang) {      this.data.set("currentLang", lang)      loci18n("body")      this.highlightCurrentLang()    }  }

Getterочень прост - мы получаем значение используемого в данный момент языка ивозвращаем его.

Setter сложнее. Преждевсего, мы используем метод changeLanguage, если текущий заданный язык не равенвыбранному. Кроме того, мы сохраняем вновь выбранную локаль податрибутом data-current-lang (к которому обращаются в getter), локализуя телоHTML-страницы с использованием плагина loc-i18next и, наконец, выделяя текущуюиспользуемую локаль.

Давайте укажем highlightCurrentLang():

  highlightCurrentLang() {    this.switcherTargets.forEach((el, i) => {      el.classList.toggle("current", this.currentLang === el.getAttribute("data-lang"))    })  }

Здесь мы повторяем по множеству языковых коммутаторов исравниваем значения их атрибутов data-lang со значением используемого в данныймомент языка. Если значения совпадают, коммутатор назначается текущим классомCSS, иначе этот класс будет удален.

Чтобы сделать конструкцию this.switcherTargets, нам нужноопределить целевые показатели стимула следующим образом:

static targets = [ "switcher" ]

Кроме того, добавьте атрибуты data-target со значениями switcher для lis:

  initialize() {      // ...      this.element.innerHTML = languages.map((lang) => {        return `<li data-action="click->languages#switchLanguage"        data-target="languages.switcher" data-lang="${lang.code}">${lang.title}</li>`      }).join("")      // ...  }

Еще одна важная вещь, которую следует учитывать, заключаетсяв том, что для перевода файлов может потребоваться некоторое время, и мы должныдождаться завершения этой операции, прежде чем разрешить коммутацию локали.Поэтому давайте воспользуемся loaded обратным вызовом:

  initialize() {    i18n.on("loaded", (loaded) => { // <---      let languages = [        {title: "English", code: "en"},        {title: "Русский", code: "ru"}      ]      this.element.innerHTML = languages.map((lang) => {        return `<li data-action="click->languages#switchLanguage"        data-target="languages.switcher" data-lang="${lang.code}">${lang.title}</li>`      }).join("")      this.currentLang = i18n.language    })  }

Наконец, не забудьте удалить setTimeout из метода loadUsers():

  loadUsers() {    fetch(this.data.get("url"))    .then(response => response.text())    .then(json => {      this.renderUsers(json)      this.localize()    })  }

Сохранение языка в URL-адресе

После переключения языкового стандарта я хотел бы добавитьпараметр GET ?Lang в URL, содержащий код выбранного языка. Добавление параметраGET без перезагрузки страницы можно легко сделать с помощью API истории:

  set currentLang(lang) {    if(i18n.language !== lang) {      i18n.changeLanguage(lang)      window.history.pushState(null, null, `?lang=${lang}`) // <---    }    if(this.currentLang !== lang) {      this.data.set("currentLang", lang)      loci18n("body")      this.highlightCurrentLang()    }  }

Обнаружение локали

Последнее, что мы собираемся реализовать сегодня, - этовозможность установить локаль на основе предпочтений пользователя. Плагин подназванием LanguageDetector может помочь нам решить эту задачу. Добавить новый Yarn пакет:

yarn add i18next-browser-languagedetector

Импорт LanguageDetector внутри файла i18n / config.js:

import LngDetector from "i18next-browser-languagedetector"

Теперь настройте конфигурацию:

const i18n = i18next.use(I18nXHR).use(LngDetector).init({ // <---  // other options go here...  detection: {    order: ["querystring", "navigator", "htmlTag"],    lookupQuerystring: "lang",  }}, function(err, t) {  if (err) return console.error(err)});

В опции orderперечислены все методы (отсортированные по их значению), которые плагин долженпопробовать, чтобы «угадать» предпочтительный язык:

  • querystring означает проверку параметра GET, содержащего кодлокали.
  • lookupQuerystring задает имя используемого параметра GET,что в нашем случае есть lang.
  • navigator означает получение данных локали по запросупользователя.
  • htmlTag включает выбор предпочтительной локали из атрибутаlang html tag.

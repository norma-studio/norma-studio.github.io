Ваше приложение станет гораздо интереснее использовать, если вы правильно задействуете в нем анимацию. Современные браузеры способны анимировать различные элементы на веб-странице, используя только CSS. Если вы используете AngularJS в одном из ваших проектов, вы можете также использовать модуль ngAnimate, чтобы добавить анимацию к нему в Angular way.

Модуль ngAnimate позволяет добавлять в ваши проекты CSS, а также анимации на основе JavaScript. Анимации на основе CSS включают ключевые кадры и переходы. Анимации на основе JavaScript необходимо регистрировать с помощью module.animation(). В этом учебном пособии вы узнаете, как добавлять анимации CSS в встроенные анимационные системы AngularJS, чтобы сделать ваши приложения более привлекательными.

Основы

Первое, что вам нужно сделать, чтобы добавить анимацию, - это включить angule-animate.js в ваш проект. После этого вы можете включить модуль ngAnimate в качестве зависимости в своем приложении, чтобы начать анимацию различных элементов. 

angular.module("app", ["ngAnimate"]);

Ниже приведен список различных директив анимации и анимационных событий, которые вы можете использовать:

  • ng-repeat может обнаруживать события входа, выхода и перемещения.
  • ng-view, ng-include, ng-switch и ng-if могут обнаруживать события входа и выхода.
  • ng-show, ng-hide и ng-class могут обнаруживать события добавления и удаления.
  • ng-message может также обнаруживать события входа и выхода.

Вы должны помнить, что эти анимационные хуки по умолчанию не включены; Они становятся активными только после добавления модуля ngAnimate в ваше приложение.

Анимация ng-if с использованием переходов CSS3

Директива ng-if в AngularJS используется для добавления или удаления элементов из DOM на основе значения данного выражения. Когда выражение принимает значение false, элемент удаляется из DOM. Когда выражение принимает значение true, клонированный элемент повторно вставляется в DOM. Это добавление и удаление элементов происходит мгновенно. Однако вы можете анимировать элементы, которые удаляются или воссоздаются с помощью модуля ngAnimate.

Если элементы добавляются в DOM, вы можете использовать классы ng-enter и ng-enter-active для их анимации. Код перехода, а также начальное состояние анимации добавляются в класс ng-enter. Конечное состояние анимации добавляется как класс ng-enter-active. Вот код, чтобы показать некоторые красные полосы, оживляя их высоту.

.red-bar.ng-enter {  transition:all ease-in 0.25s;  height: 0px;}.red-bar.ng-enter.ng-enter-active {  height: 30px;}

Аналогично, вы можете также применить анимацию, чтобы скрыть полосы, используя классы ng-leave и ng-leave-active. Я оставил анимацию для удаления красных полос, чтобы вы могли четко видеть разницу между анимированными и неанимированными состояниями.

Анимация ng-view с использованием переходов CSS3

Директива ng-view в AngularJS используется в качестве контейнера для перезагрузки части страницы, когда пользователь переключается между различными представлениями.

Также как добавление и удаление элементов с помощью ng-if, этот процесс также происходит очень быстро. Если вы хотите анимировать появление и уход различных элементов в представление, вы можете использовать те же старые классы ng-enter, ng-enter-active, ng-leave и ng-leave-active.

Помимо этих четырех классов существует также класс ng-animate. Во время анимации этот класс также добавляется ко всем элементам, которые необходимо анимировать. Вы можете использовать его для предоставления всех правил CSS, которые должны быть применимы в ходе анимации. Вот пример:

.planet-view.ng-animate {  transition: all ease 0.4s;  position: absolute;}.planet-view.ng-enter {  top: 200px;  opacity: 0;}.planet-view.ng-leave, .planet-view.ng-enter.ng-enter-active {  top: 0;  opacity: 1;}.planet-view.ng-leave.ng-leave-active {  top: -200px;  opacity: 0;}

Как вы можете видеть в приведенном выше коде, для всех свойств была применена длительность перехода 0,4 секунды. Когда информация для конкретной планеты входит в представление, она начинается с непрозрачности на нуле и положения top, установленного на 200 пикселей. В то же время элементы, которые собираются покинуть вид, имеют прозрачность 1.

К концу перехода появляющиеся элементы достигают верхнего положения представления и становятся полностью непрозрачными. Точно так же уходящие элементы переходит на позицию 200px и исчезают. Это дает нам приятный эффект, когда информация появляется в нижней части страницы и исчезает в верхней части страницы.

В следующей демонстрации ng-view попробуйте щелкнуть имя разных планет, чтобы увидеть enter и leave анимацию в действии.

Информация об атмосферном составе земных планет взята с этой страницы.

Анимация ng-repeat используя кейфрейм анимацию

Директива ng-repeat в AngularJS используется для создания экземпляра шаблона один раз для каждого элемента в заданной коллекции. Эта директива используется, когда вы пытаетесь сортировать, фильтровать, добавлять или удалять элементы из коллекции. Существует три события анимации, которые могут быть активированы с помощью ng-repeat.

  • enter: это событие срабатывает всякий раз, когда вы добавляете новый элемент в список, или элемент в списке открывается после применения фильтра.
  • leave: Это событие запускается всякий раз, когда вы удаляете элемент из списка или элемент из списка отфильтровывается.
  • move: Это событие запускается, когда смежный элемент отфильтровывается, и соответствующий элемент нужно переупорядочить.

В последних двух разделах вы использовали CSS-переходы для анимации различных элементов. В этом разделе вы узнаете, как анимировать различные элементы с помощью анимации ключевых кадров. Я предоставлю код для перехода и анимации ключевого кадра, чтобы вы могли видеть разницу между ними.

Вот код перехода, чтобы анимировать элементы:

.task-item.ng-enter,.task-item.ng-leave {  transition:all linear 0.25s;}.task-item.ng-leave {  top: 0;  opacity: 1;}.task-item.ng-leave.ng-leave-active {  top: 200px;  opacity: 0;}.task-item.ng-enter{  opacity:0;  top: -500px;}.task-item.ng-enter.ng-enter-active {  opacity:1;  top: 0px;}

Как вы видите, этот код очень похож на код перехода, который мы использовали до этого момента. Начальные состояния определены в ng-enter и ng-leave. Конечные состояния определены в ng-enter-active и ng-leave-active.

Следующий код воссоздает тот же эффект, используя анимацию ключевых кадров.

@keyframes added {  from {    opacity: 0;    top: -500px;  }  to {    opacity: 1;    top: 0px;  }}@keyframes deleted {  from {    top: 0;    opacity: 1;  }  to {    top: 200px;    opacity: 0;  }}.task-item.ng-enter {  animation: 0.25s added;}.task-item.ng-leave {  animation: 0.25s deleted;}

На этот раз мы используем свойство CSS animation для добавления анимации. Действительная анимация определяется с помощью @keyframes. Начальное и конечное состояние различных элементов было определено в самих ключевых кадрах. Обратите внимание, что нам больше не нужно использовать классы ng-enter-active и ng-leave-active. Одним из преимуществ использования ключевых кадров является то, что теперь анимация может быть намного сложнее простого перехода.

Использование ngAnimate вместе ngMessages

Envato Tuts + также опубликовал учебник по валидации ваших форм с помощью ngMessages. В последнем разделе этого руководства вы узнали, как отображать сообщения об ошибках только после того, как пользователь действительно коснулся поля ввода. Все сообщения об ошибках в этом учебнике появляются и исчезают мгновенно.

Если вы хотите добавить элегантную анимацию, чтобы отображать или скрывать сообщения об ошибках, вы можете сделать это с помощью модуля ngAnimate. Как уже упоминалось в начале этого руководства, ng-message может обнаруживать события входа и выхода. Это означает, что вы можете использовать классы ng-enter и ng-enter-active, чтобы указать начальное и конечное состояние сообщений об ошибках, отображаемых в настоящее время пользователям. Аналогичным образом вы можете использовать классы ng-leave и ng-leave-active, чтобы указать начальное и конечное состояние сообщений об ошибках, которые находятся в процессе скрытия от пользователей.

Ниже приведен пример изменения свойства непрозрачности для сообщений об ошибках.

.error-msg.ng-enter {  transition: 0.5s linear all;  opacity: 0;}.error-msg.ng-enter.ng-enter-active {  opacity: 1;}

Я добавил класс error-msg ко всем сообщениям об ошибках, и вы можете добавить свои собственные классы и соответствующим образом обновить код. В этом случае я не применял анимацию сообщений об ошибках, которые стали недействительными и больше не должны отображаться пользователям. Это означает, что они исчезнут мгновенно. Вот демонстрация, которая показывает, как использовать ngMessages и ngAnimate вместе.

В последнем разделе рассказывается, как использовать модуль ngAnimate с другими модулями, такими как ngMessages, для добавления симпатичных анимаций в сообщения об ошибках.

Одна вещь, которую вы должны иметь в виду при использовании анимации, заключается в том, что вы не должны злоупотреблять ими. Слишком много анимаций могут отвлекать пользователей от основной функции приложения. Используйте анимацию только в том случае, если вы считаете, что она действительно сможет улучшить пользовательский интерфейс.

В этом уроке мы сосредоточимся на организации процесса разработки. В частности, мы будем использовать полезные службы перехвата команд GitHub для того, чтобы автоматически обновлять состояние проекта на нашем собственном сервере каждый раз, когда мы загружаем обновления в репозиторий GitHub.

Предпочитаете обучающее видео?

Шаг 1 - Создание Git репозитория

Ах да, нам ведь понадобится какое-то подобие проекта, с которым мы будем экспериментировать. Ну давайте прямо сейчас его и сделаем. Используя любой инструмент на ваш вкус (я бы порекомендовал Structurer), создайте новую директорию, назовите ее awesomeProject и поместите в нее файл index.html. Пока что можно этот файл спокойно наполнить какой-нибудь отсебятиной.

После создания нашей тестовой директории, давайте создадим наш первый Git коммит.

Если вы еще не знакомы с Git, я очень рекомендую вам сперва ознакомиться с материалом "Простота контроля версий с Git"

Откройте терминал: 

cd path/to/awesomeProjectgit initgit add .git commit -m "First commit"

Тем из вас, кто знаком с Git, это все должно быть знакомо, как дом родной. Мы создаем Git репозиторий, добавляем все файлы в область staging, и после этого создаем наш первый коммит.

Шаг 2 - Загрузка на GitHub

Следующим шагом будет загрузка нашего проекта на GitHub. Благодаря этому, мы сможем легко выполнить команду git pull для скачивания этого проекта на какой угодно компьютер или сервер.

Опять же, если вы не знакомы с GitHub и еще не создали себе там учетную запись, прочитайте "Терминал, Git и GitHub для простых смертных".

Начнем с создания нового Git репозитория

Далее вам необходимо указать дополнительную информацию о вашем проекте. Это просто:

И в завершение, поскольку мы работаем с уже существующим на нашем компьютере Git репозиторием, все что нам нужно - это выполнить следующие команды у себя на компьютере: 

git remote add origin git@github.com:Your-Username/awesomeProject.gitgit push -u origin master

После выполнения всех вышеперечисленных шагов, наш awesomeProject станет доступен на GitHub. Это было просто!

Шаг 3 - SSH

Теперь нам, конечно же, нужен какой-нибудь способ просмотра нашего проекта в реальном времени, в идеале проект должен находиться на нашем собственном сервере. Но иногда это весьма затруднительно организовать. Понадобилось бы отправить изменения в вашем проекте на GitHub, затем авторизоваться на вашем сервере, вручную загрузить туда обновленную директорию, и так далее. Конечно, это иногда может занять незначительное время, но когда речь идет о множестве изменений в проекте в течение дня, то подобный процесс быстро может стать обременительным.

Но обо всем по порядку. Мы решим эту проблему на шаге 4. А пока давайте просто загрузим наш Git репозиторий на наш сервер. Для этого нам необходимо установить с сервером соединение по SSH.

В зависимости от вашего хостинг-провайдера, параметры SSH могут немного отличаться. Погуглив "название-вашего-хостинга SSH", вы наверняка найдете необходимые инструкции. Как разберетесь с этим, давайте продолжать:

Будем использовать мой личный сервер в качестве примера: 

ssh jeffrey-way.com@jeffrey-way.com<enter your host password>

И после этих двух команд мы подключились к серверу!

Далее переходим командой cd в директорию, внутри которой мы хотим разместить awesomeProject. В моем случае команда будет такая: cd domains/demo.jeffrey-way.com/html/. Вы, конечно, измените команду в соответствие с вашей структурой директорий.

Git Clone

Теперь давайте клонируем наш GitHub репозиторий. 

git clone git@github.com:Your-User-Name/awesomeProject.git

Несколько секунд на выполнение этой команды, и вот эта директория стала доступна на вашем сервере, а в моем случае ее содержимое доступно по адресу: http://demo.jeffrey-way.com/awesomeProject.

Шаг 4 - Привязка к репозиторию

На данном этапе основной проблемой является то, что между нашим GitHub репозиторием и директорией, которая хранится на нашем сервере, особо не существует связи - по крайней мере не в автоматическом режиме. Например, если мы обновим исходные файлы на своем компьютере и затем отправим эти изменения на GitHub командой push: 

git add index.htmlgit commit -m "Added photo of dancing chicken"git push origin master

То эти изменения никак не отразятся на нашем сервере. А с какой бы это радости?! Для того, чтобы это произошло, нам нужно снова установить SSH соединение с сервером, зайти командой cd в директорию awesomeProject, и выполнить еще одну команду git pull, чтобы загрузить обновленные исходные файлы.

Согласитесь, было бы круто, если бы каждый раз, как мы отправляем обновления на GitHub, исходные файлы автоматически обновлялись бы на нашем сервере.

Оказывается, мы можем это вполне легко реализовать с помощью служб перехвата команд GitHub.

Во можете перейти на страницу настроек перехвата команд, кликнув на вкладке "Settings" (раньше это была кнопка "Admin") в вашем GitHub репозитории, а затем кликнув "Webhooks" (раньше это был пункт "Service Hooks"). Добавление нового webhook с отметкой "Just the push event" сообщит GitHub, что необходимо отсылать запрос POST по указанному адресу каждый раз, когда вы выполняете отправку изменений на ваш репозиторий GitHub командой push. (Раньше эта настройка была вынесена в пункт Post-Receive URL(s) в Service Hooks) Это именно то, что нам нужно!

"Мы будем посылать запросы POST на указанные URL вместе и информацией об отправленных командой push изменениях."

Чтобы такой подход сработал, нам необходимо создать еще один файл, который обеспечит выполнение команды git pull. Создайте новый файл, назовите его github.php (или как пожелаете - может как-то более конкретно), и добавьте в него следующее: 

<?php `git pull`;

Вы наверное подумали: "Джефф, да ты спятил. Нельзя ведь использовать Bash скрипт в строке PHP кода." Ну... вообще как бы можно, надеюсь вы заметили, что в моем коде не просто одинарные кавычки, а именно обратные кавычки.

Когда вы заключаете последовательность символов в обратные кавычки в коде PHP, то эта последовательность символов будет воспринята как Bash скрипт. По сути это тоже самое, что и использование функции bash_exec.

Сохраните этот файл и загрузите его в директорию awesomeProject на вашем сервере. После этого, скопируйте URL к этому файлу и вставьте его в поле "Payload URL" (раньше это было поле "Post-Receive URL"). В моем случает URL выглядит так: http://demo.jeffrey-way.com/awesomeProject/github.php.

С такими настройками, каждый раз, когда вы отправляете изменения командой push на ваш репозиторий GitHub, будет вызываться этот файл, и директория awesomeProject на вашем сервере будет автоматически обновляться, а вам не придется и пальцем шевелить для этого. Вполне себе недурственно, согласны?

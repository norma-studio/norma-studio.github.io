Когда вы самостоятельно размещаете WordPress, получение контактной формы может быть одной из наиболее запутанных задач. Форма контакта в вашей теме обычно не является проблемой - она настраивает исходящий почтовый сервер. Я очень не рекомендую людям запускать почтовый сервер самостоятельно. Есть много причин не делать этого, особенно по причине производительности и безопасности. К счастью, есть простое решение - бесплатный плагин Mailgun для WordPress.

Использование плагина Mailgun для WordPress обеспечивает простое бесплатное решение для предотвращения проблем с запуском собственного почтового сервера или преодолением ограничений почтового сервера вашего хоста. Это упрощает получение контактных форм.

Примечание. Расходы Mailgun основаны на объеме электронной почты, но он бесплатен до многих тысяч электронных писем в месяц - достаточно для большинства контактных форм WordPress.

Установка плагина Mailgun

На панели инструментов WordPress откройте плагины -> Добавить новый. Найдите плагин Mailgun for WordPress и установите его.

Хотя вам и не нужно это понимать, Mailgun представит вам примеры того, как отправлять электронную почту с помощью своей службы, используя cURL, Ruby, Python, PHP и др. Вы можете использовать это, чтобы проверить службу Mailgun, если хотите.

Вам не нужно добавлять домен для использования Mailgun с WordPress. Сообщения, которые поступают из вашей контактной формы, будут доставлены вам с адреса электронной почты, такого как sandbox2732dxxxxxxxyyyyyyyjzzzzz49.mailgun.org. Посетители вашего сайта не будут видеть этот адрес. Когда вы отвечаете от своего почтового клиента, он должен исходить из вашего основного почтового адреса.

Вернитесь на страницу настроек WordPress Dashboard, Mailgun Plugin и предоставите тестовый домен и ключ API, предоставленные вам с панели управления Mailgun:

Любой плагин темы или контактной формы, который использует стандартные API электронной почты WordPress (например, wp_mail), должен автоматически работать с Mailgun. Попробуйте его со своей контактной формой WordPress - в большинстве случаев она будет работать без проблем.

Если у вас нет контактной формы в вашей теме или на сайте, то мне нравится Контактная форма 7 (см. также «Оптимизация контактной формы 7»).

Mailgun затем потребует, чтобы вы сделали изменения DNS везде, где вы зарегистрировали свое доменное имя. Вам нужно будет создать текстовые записи для своего домена, как показано ниже:

Запись TXT корневого имени хоста может быть настроена с помощью знака @, например. @ TXT => v=spf1 include:mailgun.org ~all. Другая TXT-запись настроена как k1._domainkey.yourdomain.com TXT => k = rsa; ...... здесь длинная строка ...

Примечание. Некоторые регистраторы доменных имен имеют проблемы с длинными строками и специальными символами, необходимыми для этого. Раньше у меня были большие проблемы с MyDomain.com. Теперь я использую NameCheap.com и с этим проблем больше нет.

Другое использование Mailgun

Я большой энтузиаст Mailgun. Его можно использовать для многих целей, как только вы настроите собственное доменное имя. Например:

  • Используйте безопасные SMTP-серверы Mailgun для отправки всей исходящей электронной почты.
  • Настройте записи MX для своего домена, чтобы фильтры спама Mailgun предварительно отображали всю вашу электронную почту.
  • Используйте API-адрес почтового ящика Mailgun, как я описал в этом учебнике с открытым исходным кодом. Это может использоваться как замена для PHP List, MailChimp или другого маркетингового сервиса.
  • Используйте API Mailgun для программной отправки электронной почты для вашего приложения.
  • Используйте Mailgun как часть фильтрации, организации и дайджест решения для вашей электронной почты.

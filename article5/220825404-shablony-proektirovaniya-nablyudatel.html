Интро

Сегодня мы рассмотрим еще один поведенческий шаблон, который называется Наблюдатель. Название Наблюдатель означает, что кто-то наблюдает за вашими действиями и может случится так, что наблюдатель будет выполнять какие-либо действия в зависимости от вашей активности. ...

Сегодня мы рассмотрим еще один поведенческий шаблон, который называется Наблюдатель. Название Наблюдатель означает, что кто-то наблюдает за вашими действиями и может случится так, что наблюдатель будет выполнять какие-либо действия в зависимости от вашей активности.

Такая концепция применима к этому шаблону проектирования. Следует реализовывать этот шаблон, когда у нас имеется зависимость один-ко-многим между нашим объектом и другими объектами, которые должны быть изменены или уведомлены при любых изменениях любого другого объекта.

Wikipedia говорит о шаблоне почти тоже самое:

Шаблон наблюдатель - это шаблон проектирования программного обеспечеия, в котором объект, называемый подчиненный, управляет списком зависимостей, называемых наблюдателями, и уведомляет их автоматически о любых изменениях состояния, обычно вызывая один из их методов. В основном шаблон используется для реализации распределенных систем управления событиями. Шаблон Наблюдатель также является ключевой частью широко известного архитектурного шаблона модель-отображение-контроллер (MVC).

Чтобы более детально рассмотреть шаблон, я возьму для примера имитатор, который показывает различные значения валют по отношению к доллару. Мы предполагаем, что имитатор показывает цену и так же обновляет цену в один и тот же интервал.

Перед тем как мы начнем, следует определиться с обязанностями класса симулятора (который ведет себя как наблюдатель в этом примере).

  1. Наблюдатель должен иметь возможность добавлять новую валюту, так чтобы клиент мог добавить столько валют, сколько ему необходимо.
  2. Наблюдатель должен иметь связь со всеми добавленными валютами.
  3. Наблюдатель должен показывать статус/значение каждой зарегистрированной валюты.

В следующем разделе мы реализуем нашего наблюдателя:

interface Observer {    public function addCurrency(Currency $currency);}class priceSimulator implements Observer {  private $currencies;public function __construct() {$this->currencies = array();}public function addCurrency(Currency $currency) {array_push($this->currencies, $currency);}public function updatePrice() {foreach ($this->currencies as $currency) {$currency->update();}}}

Если вы взглянете на код выше, то увидите, что в нем реализованы все обязанности, которые мы определили раньше.

Теперь наш наблюдатель готов. Что требуется сейчас, так это несколько разных валют, которые мы можем добавить к наблюдателю. Давайте реализуем классы валют.

interface Currency {    public function update();public function getPrice();}class Pound implements Currency {private $price;public function __construct($price) {$this->price = $price;echo "<p>Pound Original Price: {$price}</p>";}public function update() {$this->price = $this->getPrice();echo "<p>Pound Updated Price : {$this->price}</p>";}public function getPrice() {return f_rand(0.65, 0.71);}}class Yen implements Currency {private $price;public function __construct($price) {$this->price = $price;echo "<p>Yen Original Price : {$price}</p>";}public function update() {$this->price = $this->getPrice();echo "<p>Yen Updated Price : {$this->price}</p>";}public function getPrice() {return f_rand(120.52, 122.50);}}

Сейчас у нас все готово, чтобы запустить нашего наблюдателя.

Собирая Все Вместе

interface Observer {    public function addCurrency(Currency $currency);}class priceSimulator implements Observer {private $currencies;public function __construct() {$this->currencies = array();}public function addCurrency(Currency $currency) {array_push($this->currencies, $currency);}public function updatePrice() {foreach ($this->currencies as $currency) {$currency->update();}}}interface Currency {public function update();public function getPrice();}class Pound implements Currency {private $price;public function __construct($price) {$this->price = $price;echo "<p>Pound Original Price: {$price}</p>";}public function update() {$this->price = $this->getPrice();echo "<p>Pound Updated Price : {$this->price}</p>";}public function getPrice() {return f_rand(0.65, 0.71);}}class Yen implements Currency {private $price;public function __construct($price) {$this->price = $price;echo "<p>Yen Original Price : {$price}</p>";}public function update() {$this->price = $this->getPrice();echo "<p>Yen Updated Price : {$this->price}</p>";}public function getPrice() {return f_rand(120.52, 122.50);}}function f_rand($min=0,$max=1,$mul=1000000){if ($min>$max) return false;return mt_rand($min*$mul,$max*$mul)/$mul;}$priceSimulator = new priceSimulator();$currency1 = new Pound(0.60);$currency2 = new Yen(122);$priceSimulator->addCurrency($currency1);$priceSimulator->addCurrency($currency2);echo "<hr />";$priceSimulator->updatePrice();echo "<hr />";$priceSimulator->updatePrice();

Код выше выведет:

Pound Original Price: 0.6Yen Original Price : 122-------------Pound Updated Price : 0.65346Yen Updated Price : 121.287809-------------Pound Updated Price : 0.671269Yen Updated Price : 121.300605

Здесь видно, что мы обновили цены всех зарегистрированных валют и они были показаны в нашем симуляторе. Теперь нам следует разобраться, как мы можем добавить новые валюты к симулятору с минимальными изменениями.

Эти изменения будут включать в себя только регистрацию валют в симуляторе. Таким образом наш клиентский код остается нетронутым, всякий раз, когда он вызывает обновление цен в нашем симуляторе.

Довольно легко и просто добавить новую валюту. Сейчас все что нам нужно сделать, это зарегистрировать новую валюту в нашем наблюдателе.

$priceSimulator = new priceSimulator();$currency1 = new Pound(0.60);$currency2 = new Yen(122);$currency3 = new Euro(122);$priceSimulator->addCurrency($currency1);$priceSimulator->addCurrency($currency2);$priceSimulator->addCurrency($currency3);echo "<hr />";$priceSimulator->updatePrice();echo "<hr />";$priceSimulator->updatePrice();

Заключение

Состояние любого объекта имеет большое значение в мире объектно-ориентированного программирования, потому что все процессы протекают между объектами и их взаимодействиями друг с другом. И часто бывает так, что следует уведомить несколько объектов об изменениях, которые произошли в других объектах. Шаблон проектирования Наблюдатель может быть использован, когда при изменении состояния одного объекта все зависящие от него оповещаются об этом событии.

Не забудьте оставить обратную связь.

Когда я впервые услышал о Node.js, я подумал, что это просто реализация JavaScript для сервера. Но на самом деле это намного больше: в нем есть множество встроенных функций, которые вы не получаете в браузере. Одним из таких функциональных возможностей является модуль событий, который имеет класс EventEmitter. Мы рассмотрим его в этом уроке.

EventEmitter: Что и почему

Последний довод в пользу событий: они являются очень свободным способом соединения частей вашего кода вместе.

Итак, что именно делает класс EventEmitter? Проще говоря, он позволяет прослушивать «события» и назначать действия для запуска, когда происходят эти события. Если вы знакомы с интерфейсом JavaScript, вы узнаете о событиях мыши и клавиатуры, которые происходят в определенных пользовательских взаимодействиях. Они очень похожи, за исключением того, что мы можем генерировать события самостоятельно, когда хотим, и не обязательно, основываясь на взаимодействии с пользователем. Принципы EventEmitter основаны на том, что они назывались моделью публикации/подписки, потому что мы можем подписаться на события, а затем опубликовать их. Существует много интерфейсных библиотек, построенных с поддержкой pub/sub, но Node имеет встроенный интерфейс.

Другой важный вопрос заключается в следующем: почему вы использовали модель событий? В node это альтернатива глубоко вложенным обратным вызовам. Многие методы node выполняются асинхронно, а это значит, что для запуска кода после завершения метода вам необходимо передать метод обратного вызова функции. В конце концов, ваш код будет похож на гигантскую воронку. Чтобы предотвратить это, многие классы node генерируют события, которые вы можете слушать. Это позволяет упорядочить код так, как вам бы хотелось, и не использовать обратные вызовы.

Последнее помогает событиям: они являются очень свободным способом соединения частей вашего кода вместе. Событие может быть запущено, но если его никто не прослушивает, все в порядке: оно просто останется незамеченным. Это означает, что удаление слушателей (или генерация событий) никогда не приводит к ошибкам JavaScript.

Использование EventEmitter

Мы начнем с класса EventEmitter. До него довольно просто добраться: нам просто нужен модуль событий:

    var events = require("events");

Этот объект events имеет одно свойство, которое является классом EventEmitter. Итак, давайте сделаем простой пример для начинающих:

    var EventEmitter = require("events").EventEmitter;    var ee = new EventEmitter();    ee.on("someEvent", function () {        console.log("event has occured");    });    ee.emit("someEvent");

Начнем с создания нового объекта EventEmitter. Этот объект имеет два основных метода, которые мы используем для событий: on и emit.

Мы начнем с on. Этот метод принимает два параметра: мы начинаем с имени события, которое мы слушаем: в этом случае это «someEvent». Но, конечно, это может быть что угодно, и вы обычно выбираете что-то лучше. Второй параметр - это функция, которая будет вызываться при возникновении события. Это все, что требуется для настройки события.

Теперь, чтобы запустить событие, вы передаете имя события методу ement метода EventEmitter. Это последняя строка кода выше. Если вы выполните этот код, вы увидите, что мы распечатаем текст на консоли.

Это самое основное использование EventEmitter. Вы также можете включать данные при создании событий: 

    ee.emit("new-user", userObj);

Это только один параметр данных, но вы можете включить столько, сколько хотите. Чтобы использовать их в функции обработчика событий, просто возьмите их в качестве параметров:

    ee.on("new-user", function (data) {        // use data here    });

Прежде чем продолжить, позвольте мне пояснить часть функциональности EventEmitter. Мы можем иметь более одного слушателя для каждого события; может быть назначено несколько прослушивателей событий (все с помощью on), и все функции будут вызываться при запуске события. По умолчанию Node позволяет одновременно принимать до десяти слушателей на одном событии; если будет создано больше, node выдаст предупреждение. Однако мы можем изменить эту сумму, используя setMaxListeners. Например, если вы запустите это, вы увидите предупреждение, выводимое выше вывода:

    ee.on("someEvent", function () { console.log("event 1"); });    ee.on("someEvent", function () { console.log("event 2"); });    ee.on("someEvent", function () { console.log("event 3"); });    ee.on("someEvent", function () { console.log("event 4"); });    ee.on("someEvent", function () { console.log("event 5"); });    ee.on("someEvent", function () { console.log("event 6"); });    ee.on("someEvent", function () { console.log("event 7"); });    ee.on("someEvent", function () { console.log("event 8"); });    ee.on("someEvent", function () { console.log("event 9"); });    ee.on("someEvent", function () { console.log("event 10"); });    ee.on("someEvent", function () { console.log("event 11"); });    ee.emit("someEvent");

Чтобы установить максимальное количество слушателей, добавьте эту строку над слушателями:

    ee.setMaxListeners(20);

Теперь, когда вы запустите его, вы не получите предупреждение.

Другие методы EventEmitter

Есть несколько других методов EventEmitter, которые вы найдете полезными.

Вот еще один: once. Он похож на метод on, за исключением того, что он работает только один раз. После вызова в первый раз слушатель удаляется.

    ee.once("firstConnection", function () { console.log("You"ll never see this again"); });    ee.emit("firstConnection");    ee.emit("firstConnection");

Если вы запустите это, вы увидите сообщение только один раз. Второе излучение события не подхвачено ни одним слушателем (и это нормально, кстати), потому что once прослушиватель был удален после того, как его использовали один раз.

Говоря об удалении слушателей, мы можем сделать это сами, вручную, несколькими способами. Во-первых, мы можем удалить слушателя с помощью метода removeListener. Он принимает два параметра: имя события и функцию слушателя. До сих пор мы использовали анонимные функции в качестве наших слушателей. Если мы хотим удалить слушателя позже, он должен быть функцией с именем, на которое мы можем ссылаться. Мы можем использовать этот метод removeListener для дублирования эффектов метода once:

    function onlyOnce () {        console.log("You"ll never see this again");        ee.removeListener("firstConnection", onlyOnce);    }    ee.on("firstConnection", onlyOnce)     ee.emit("firstConnection");    ee.emit("firstConnection");

Если вы запустите это, вы увидите, что он имеет тот же эффект, что и once.

Если вы хотите удалить всех слушателей, связанных с данным событием, вы можете использовать removeAllListeners; просто передайте ему название события:

    ee.removeAllListeners("firstConnection");

Чтобы удалить всех слушателей для всех событий, вызовите функцию без каких-либо параметров.

ee.removeAllListeners();

Есть один последний метод: listener. Этот метод принимает имя события в качестве параметра и возвращает массив всех функций, которые прослушивают это событие. Вот пример этого, основанный на нашем примере onlyOnce:

    function onlyOnce () {        console.log(ee.listeners("firstConnection"));        ee.removeListener("firstConnection", onlyOnce);        console.log(ee.listeners("firstConnection"));    }    ee.on("firstConnection", onlyOnce)     ee.emit("firstConnection");    ee.emit("firstConnection");

Мы закончим этот раздел с одной битностью. Наш экземпляр EventEmitter фактически запускает два собственных события, которые мы можем слушать: один, когда мы создаем новых слушателей, и один, когда мы удаляем их. Смотрим сюда:

    ee.on("newListener", function (evtName, fn) {        console.log("New Listener: " + evtName);    });    ee.on("removeListener", function (evtName) {        console.log("Removed Listener: " + evtName);    });    function foo () {}    ee.on("save-user", foo);    ee.removeListener("save-user", foo);

Запустив это, вы увидите, что наши слушатели как для новых слушателей, так и для удаленных слушателей были запущены, и мы получаем сообщения, которые мы ожидали.

Итак, теперь, когда мы видели все методы, которые имеет экземпляр EventEmitter, давайте посмотрим, как он работает вместе с другими модулями.

EventEmitter внутри других модулей

Поскольку класс EventEmitter является просто обычным JavaScript, имеет смысл, что он может использоваться в других модулях. Внутри ваших собственных модулей JavaScript вы можете создавать экземпляры EventEmitter и использовать их для обработки внутренних событий. Это просто. Более интересно было бы создать модуль, который наследуется от EventEmitter, поэтому мы можем использовать его функциональную часть публичного API.

На самом деле, есть встроенные модули node, которые делают именно это. Например, вы можете быть знакомы с модулем http; это модуль, который вы будете использовать для создания веб-сервера. Этот базовый пример показывает, как метод on класса EventEmitter стал частью класса http.Server:

    var http = require("http");    var server = http.createServer();    server.on("request", function (req, res) {        res.end("this is the response");    });    server.listen(3000);

Если вы запустите этот фрагмент, процесс будет ждать запроса; вы можете перейти на http://localhost:3000, и вы получите ответ. Когда экземпляр сервера получает запрос из вашего браузера, он выдает событие «request», событие, которое наш слушатель получит и сможет действовать.

Итак, как мы можем создать класс, который будет наследоваться от EventEmitter? На самом деле это не так сложно. Мы создадим простой класс UserList, который обрабатывает объекты пользователя. Итак, в файле userlist.js мы начнем с этого:

    var util         = require("util");    var EventEmitter = require("events").EventEmitter;

Нам нужен модуль util, чтобы помочь с наследованием. Далее нам нужна база данных: вместо использования реальной базы данных мы просто используем объект:

    var id = 1;    var database = {        users: [            { id: id++, name: "Joe Smith",  occupation: "developer"    },            { id: id++, name: "Jane Doe",   occupation: "data analyst" },            { id: id++, name: "John Henry", occupation: "designer"     }        ]    };

Теперь мы можем создать наш модуль. Если вы не знакомы с модулями Node, вот как они работают: любой JavaScript, который мы пишем внутри этого файла, по умолчанию доступен только изнутри файла. Если мы хотим сделать его частью общедоступного API модуля, мы сделаем его свойством module.exports или присвоим целому новому объекту или функции module.exports. Давай сделаем это:

    function UserList () {        EventEmitter.call(this);    }

Это функция конструктора, но это не ваша обычная функция JavaScript-конструктора. Здесь мы используем метод call в конструкторе EventEmitter для запуска этого метода в новом объекте UserList (который является this). Если нам нужно сделать любую другую инициализацию для нашего объекта, мы могли бы сделать это внутри этой функции, но это все, что мы сделаем на данный момент.

Недостаточно наследовать конструктор; нам также необходимо наследовать прототип. В этом случае появляется модуль util. 

    util.inherits(UserList, EventEmitter);

Это добавит все, что находится в EventEmitter.prototype к UserList.prototype; теперь наши экземпляры UserList будут иметь все методы экземпляра EventEmitter. Но мы хотим добавить еще, конечно. Мы добавим метод save, чтобы мы могли добавлять новых пользователей.

    UserList.prototype.save = function (obj) {        obj.id = id++;        database.users.push(obj);        this.emit("saved-user", obj);      };

Этот метод берет объект для сохранения в нашей «database»: он добавляет id и помещает его в массив пользователей. Затем он выдает событие «saved-user» и передает объект в виде данных. Если бы это была настоящая база данных, сохранение ее, вероятно, было бы асинхронной задачей, то есть для работы с сохраненной записью нам нужно было бы принять обратный вызов. Альтернатива этому - испустить событие, как мы это делаем. Теперь, если мы хотим что-то сделать с сохраненной записью, мы можем просто прослушать событие. Мы сделаем это через секунду. Давайте просто закроем UserList 

    UserList.prototype.all = function () {        return database.users;    };    module.exports = UserList;

Я добавил еще один метод: простой, который возвращает всех пользователей. Затем мы присваиваем UserList в module.exports.

Теперь давайте посмотрим на это; в другом файле, скажем test.js. Добавьте следующее:

    var UserList = require("./userlist");    var users = new UserList();    users.on("saved-user", function (user) {        console.log("saved: " + user.name + " (" + user.id + ")");    });    users.save({ name: "Jane Doe", occupation: "manager" });    users.save({ name: "John Jacob", occupation: "developer" });

После добавления нашего нового модуля и создания его экземпляра мы слушаем событие «saved-user». Затем мы можем продолжить и сохранить несколько пользователей. Когда мы запустим это, вы увидите, что мы получаем два сообщения, распечатываем имена и идентификаторы сохраненных нами записей.

    saved: Jane Doe (4)    saved: John Jacob (5)

Конечно, это могло бы работать наоборот: мы могли бы использовать метод on внутри нашего класса и метод emit вне или внутри, так и снаружи. Но это хороший пример того, как это можно сделать.

В этой статье я продемонстрирую основные возможности Intl и познакомлю вас с поддержкой миллиардов людей в Интернете, живущих за пределами вашей страны!

Основные функции

Пространство имен Intl охватывает три основных области функциональности:

  • Форматирование чисел
  • Форматирование дат
  • Сортировка строк

Внутри каждого из них имеются различные параметры для управления языками, используемыми для форматирования, а также параметры форматирования. Например, средство форматирования чисел включает поддержку обработки валюты. Форматер даты имеет опции для отображения частей даты.

Давайте взглянем на несколько примеров.

Добавление форматирования чисел

Следующие изменения можно увидеть в app2.js и протестировать с test2.html. Сначала я изменю свой код для вызова новой функции numberFormat:

$table.append(""+s[i].date+""+numberFormat(s[i].views)+"");

И вот эта функция:

function numberFormat(n) {        //cache the formatter once        if(window.Intl && !window.numberFormatter) window.numberFormatter = window.Intl.NumberFormat();        if(window.numberFormatter) {            return window.numberFormatter.format(n);        } else {            return n;           }    }

Функция начинается с проверки, существует ли Intl как часть объекта окна. Если это так, то мы проверяем, сделали ли мы форматировщик раньше. Объект Intl создает объект форматирования, который вы можете использовать повторно, и, поскольку мы форматируем группу чисел, мы хотим создать ее только один раз. Именно это мы и делаем, как только видим, что нам тоже нужно. На данный момент мы не заботимся о каких-либо вариантах, просто чтобы все было просто. Наконец, если вообще не было поддержки Intl, мы просто возвращаем число как есть. Результатом является значительное улучшение при минимальной работе:

Здорово! Итак, как мы тестируем другие языки? Вы можете испытать желание проверить настройки вашего браузера. Все браузеры предпочитают язык, но, к сожалению, недостаточно изменить язык в браузере. Изменение его влияет на поведение браузера.

Если вы откроете свои инструменты разработки и посмотрите на сетевые запросы, вы увидите, что заголовок под названием «Accept-Lanage» будет меняться в зависимости от ваших настроек. Например, если вы добавите французский (я предполагаю, что вы не являетесь носителем французского языка), вы увидите «fr», добавленный в этот заголовок. Но это не влияет на Intl. Вместо этого вам нужно изменить язык операционной системы и перезапустить браузер. Это не так страшно, как кажется. Когда я тестировал, я волновался, что вся операционная система немедленно изменится. Но в моих тестах я смог изменить язык, перезапустить браузер и увидеть изменения. Я быстро перешел обратно. Функции форматирования Intl позволяют переопределить текущую локаль и вместо нее передать ее.

Я изменил приложение, чтобы позволить конечному пользователю указать язык с помощью раскрывающегося списка. Вот модификация, сделанная в HTML. (Эта модификация может быть найдена в test3.html)

<select id="langDropdown">        <option value="">None Specified</option>        <option value="en-US">English (US)</option>        <option value="fr-FR">French (France)</option>        <option value="lt">Lithuanian</option>    </select>

Языки, которые я выбрал, были довольно произвольными. Далее - я обновил код своего приложения, чтобы прослушать изменения этого раскрывающегося списка, а также проверить нужный языковой стандарт при форматировании.

Cписок 3: app3.js:

function numberFormat(n) {        if(window.Intl) {            var lang = $("#langDropdown").val();            if(lang === "") lang = navigator.language;            var formatter = new window.Intl.NumberFormat(lang);            return formatter.format(n);        } else {            return n;           }    }    function getStats() {        $.getJSON("stats.json").done(function(s) {            //iterate over stats and add to table            for(var i=0; i < s.length; i++) {                $table.append(""+s[i].date+""+numberFormat(s[i].views)+"");            }        }).fail(function(e) {            console.log("we failed");            console.dir(e);        });    }    $(document).ready(function() {        //get the table dom        $table = $("#stats tbody");        //notice changes to drop down        $("#langDropdown").on("change", function(e) {            $table.empty();            getStats();        });        getStats();    });

Начиная снизу - обратите внимание, что я добавил простой обработчик событий для внесения изменений в выпадающий список. Когда обнаруживается изменение, таблица очищается и запускается функция getStats. Эта новая функция просто абстрагирует AJAX-код, использованный ранее. Настоящее изменение сейчас в numberFormat. Я проверяю выбранный язык, и если один из них был выбран, мы передаем его в качестве местного конструктора NumberFormat. Обратите внимание, что, если что-то не было выбрано, мы не выполняем своих обязательств к navigator.language. Это теперь дает нам способ быстро проверить различные места действия и видеть, как они отдают числа.

Добавление форматирования даты

Теперь прекрасное время, чтобы заботиться о другой колонке данных - числа. Я следовал за тем же стилем как прежде и добавил требование к новой функции, dateFormat.

$table.append(""+dateFormat(s[i].date)+""+numberFormat(s[i].views)+"");

И вот dateFormat (Вы можете найти кодекс в app4.js, который используется test4.html):

function dateFormat(n) {    //Used for date display    var opts = {};    opts.day = "numeric";    opts.weekday = "long";    opts.year = "numeric";    opts.month = "long";    if(window.Intl) {        var lang = $("#langDropdown").val();        if(lang === "") lang = navigator.language;        var formatter = new window.Intl.DateTimeFormat(lang, opts);        n = new Date(n);        return formatter.format(n);    } else {        return n;       }}

Это очень похоже на форматирование числа, кроме этого времени, которое мы явно передаем некоторым вариантам, когда мы создаем средство форматирования. Варианты определяют обоих, какие области видимы на дате, а также как они смотрят. Каждую часть даты можно показать или нет, и у каждого есть различные варианты. Варианты включают:

  • будний день
  • Эра
  • год
  • месяц
  • день
  • час
  • минута
  • второй
  • TimeZoneName

Полный список значений, которые можно использовать, см. В документации MDN для DateTimeFormat, но, например, месяцы могут отображаться в виде числа или в различных текстовых формах. Так что же это создает? Вот английская версия:

Покажите мне деньги

Форматирование валюты - это не отдельный объект, а необязательное использование средства форматирования чисел. Для следующей демонстрации мы создали новый файл данных stats2.json и добавили столбец «продажи» к нашим данным. Вот образец:

{"date":"4/1/2013","views":938213,"sales":3890.21},{"date":"4/2/2013","views":238213,"sales":9890.10}

Столбец был добавлен в HTML (test5.html), добавлен в JavaScript с итерацией по строкам данных (см. app5.js) и передан новой функции под названием currencyFormat. Давайте посмотрим на это.

function currencyFormat(n) {    var opts = {};    opts.style = "currency";    opts.currency = "USD";    if(window.Intl) {        var lang = $("#langDropdown").val();        if(lang === "") lang = navigator.language;        var formatter = new window.Intl.NumberFormat(lang,opts);        return formatter.format(n);    } else {        return n;       }}

Для отображения чисел в виде валют требуются два необязательных значения. Сначала стиль «валюта», а затем тип валюты. Существуют и другие варианты (например, как отображать название валюты). Вот часть, которая может вас немного поднять. Вы должны указать тип валюты.

Вы можете подумать - как, черт возьми, я могу определить тип валюты для всех возможных значений? Возможные значения основаны на спецификации (http://www.currency-iso.org/en/home/tables/table-a1.html) и теоретически вы можете анализировать их XML, но вы не хотите делать тот. Причина довольно очевидна, но я могу честно сказать, что изначально я тоже забыл. Вы не хотите просто заново отобразить конкретное число в определенной валюте. Зачем? Потому что десять долларов США, конечно, не то же самое, что десять долларов в песо. Это довольно очевидно, и, надеюсь, я единственный, кто забудет это.

Используя приведенный выше код, мы можем увидеть следующие результаты на французском языке. Обратите внимание, как числа отформатированы правильно для локали, и после номера ставится символ валюты.

Сортировка с разборки по копиям

В нашем последнем примере мы рассмотрим конструктор Collator. Collators позволяют вам сортировать текст. В то время как некоторые языки следуют простой системе заказов от A до Z, другие языки имеют другие правила. И, конечно, все становится еще интереснее, когда вы начинаете добавлять акценты. Можете ли вы сказать наверняка, если ä идет после? Я знаю, что не могу. Конструктор сборщика принимает ряд аргументов, чтобы помочь ему точно определить, как он должен сортироваться, но значение по умолчанию, вероятно, будет работать хорошо для вас.

Для этого примера мы создали совершенно новую демонстрацию, но похожую на предыдущие. В test6.html вы можете увидеть новую таблицу для студентов. Наш новый код загрузит JSON-пакет студентов и затем отсортирует их на клиенте. Данные JSON - это просто массив имен, поэтому я пропущу показ выдержки. Давайте посмотрим на логику приложения.

Листинг 4: app6.js:

function sorter(x,y) {        if(window.Intl) {            var lang = $("#langDropdown").val();            if(lang === "") lang = navigator.language;                  return window.Intl.Collator(lang).compare(x,y);        } else {            return x.localeCompare(y);          }    }    function getStudents() {        $.getJSON("students.json").done(function(s) {            //iterate over stats and add to table            s.sort(sorter);            for(var i=0; i < s.length; i++) {                $table.append(""+s[i]+"");            }        }).fail(function(e) {            console.log("we failed");            console.dir(e);        });    }    $(document).ready(function() {        //get the table dom        $table = $("#students tbody");        //notice changes to drop down        $("#langDropdown").on("change", function(e) {            $table.empty();            getStudents();        });        getStudents();    });

Как я уже сказал, этот код очень похож на предыдущие примеры, поэтому сначала остановимся на getStudents. Важная строка здесь: s.sort (сортировщик). Мы используем встроенную функцию для массивов, чтобы выполнить сортировку через пользовательскую функцию. Эта функция будет передана для сравнения двух вещей и должна вернуть -1, 0 или 1, чтобы представить, как эти два элемента должны быть отсортированы. Теперь давайте посмотрим на сортировщик.

Если у нас есть объект Intl, мы создаем новый Collator (и снова мы разрешаем вам передать локаль), а затем запускаем функцию сравнения. Вот и все. Как я уже сказал, есть варианты изменить порядок сортировки, но мы можем использовать значения по умолчанию. Резервным вариантом является localeCompare, который также пытается использовать форматирование, специфичное для локали, но имеет (в этой форме) немного лучшую поддержку. Мы могли бы также проверить эту поддержку и добавить еще один запасной вариант для действительно хорошей поддержки, но я оставлю это для вас в качестве упражнения.

Завершение

В целом, класс Intl обеспечивает некоторые очень простые способы добавить место действия определенное форматирование к Вашему кодексу. Это - конечно, что-то, что Вы можете найти теперь, вероятно, в нескольких тысячах различных библиотек JavaScript, но замечательно видеть, что производители браузера добавляют в поддержке непосредственно в самом языке. Отсутствие поддержки iOS - неприятность, но надо надеяться это будет скоро добавлено.

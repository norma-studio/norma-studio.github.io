Шаг 1: Создайте index.html

Далее приводится его содержимое:

<!DOCTYPE html><html>  <head><title>Pong</title><style>/* Removes Mobile Highlight */ *{-webkit-tap-highlight-color: rgba(0, 0, 0, 0);}</style><script src="http://code.createjs.com/easeljs-0.4.2.min.js"></script><script src="http://code.createjs.com/tweenjs-0.2.0.min.js"></script><script src="http://code.createjs.com/soundjs-0.2.0.min.js"></script><script src="http://code.createjs.com/preloadjs-0.1.0.min.js"></script><script src="http://code.createjs.com/movieclip-0.4.1.min.js"></script><script src="assets/soundjs.flashplugin-0.2.0.min.js"></script><script src="Main.js"></script></head><body onload="Main();"><canvas id="PongStage" width="480" height="320"></canvas></body></html>

Как вы видите, он довольно короткий и в основном содержит строки для скачивания библиотек CreateJS.

С момента выхода CreateJS (который по сути объединяет в набор отдельные библиотеки CreateJS) нам больше нет необходимости скачивать файлы JS и размещать их на собственном сервере; теперь файлы расположены на CDN (* content delivery network – сеть доставки (распространения) контента. Здесь и далее прим. пер.), благодаря которой мы можем скачивать эти файлы удаленно на максимальной скорости.

Давайте просмотрим код:

<style>/* Removes Mobile Highlight */ *{-webkit-tap-highlight-color: rgba(0, 0, 0, 0);}</style>

В этой строке мы убираем подсветку, которая может возникать при попытке поиграть в игру на телефоне. (Благодаря этому не будет подсвечиваться объект canvas, что происходит на телефонах при движении пальцев).

Далее идут строки для скачивания библиотек CreateJS: 

<script src="http://code.createjs.com/easeljs-0.4.2.min.js"></script><script src="http://code.createjs.com/tweenjs-0.2.0.min.js"></script><script src="http://code.createjs.com/soundjs-0.2.0.min.js"></script><script src="http://code.createjs.com/preloadjs-0.1.0.min.js"></script><script src="http://code.createjs.com/movieclip-0.4.1.min.js"></script>

Благодаря этому коду происходит скачивание файлов JS с CDN для CreateJS и у нас есть возможность использовать любые функции CreateJS в нашем коде.

Далее мы скачаем плагин Flash (* мультимедийная программная платформа компании Adobe Systems для создания веб-приложений, рекламных баннеров, анимации, игр, десктопных приложений и также для воспроизведения на веб-страницах видео- и аудиозаписей. В июле 2017 года Adobe признала технологию Flash устаревшей, её жизненный цикл будет завершён в конце 2020 года, когда Adobe прекратит поддержку и распространение Flash) SoundJS, который обеспечивает поддержку звука для браузеров, не поддерживающих HTML5 Audio. Она реализуется за счет использования SWF (* Small Web Format) (объекта Flash) для загрузки звуков.

<script src="assets/soundjs.flashplugin-0.2.0.min.js"></script>

В данном случае мы не будем использовать CDN; вместо этого мы скачаем библиотеку SoundJS с http://createjs.com/#!/SoundJS/download и поместим файлы soundjs.flashplugin-0.2.0.min.js и FlashAudioPlugin.swf в локальную библиотеку под названием assets.

Последним из файлов JS мы загрузим файл Main.js, в котором будет расположен весь код для нашей игры:

<script src="Main.js"></script>

И, наконец, давайте поместим объект Canvas на нашу сцену.

<body onload="Main();"><canvas id="PongStage" width="480" height="320"></canvas></body>

Теперь мы можем начать работать над кодом для игры.

Эта функция выполняется первой после загрузки всех файлов JS, указанных в index.html. И где она вызывается?

Шаг 4: Создание функции для реализации предварительной загрузки

function handleProgress(event){//use event.loaded to get the percentage of the loading}function handleComplete(event) {         //triggered when all loading is complete}function handleFileLoad(event) {         //triggered when an individual file completes loading                     switch(event.type)         {         case PreloadJS.IMAGE:         //image loaded          var img = new Image();              img.src = event.src;              img.onload = handleLoadComplete;              window[event.id] = new Bitmap(img);         break;         case PreloadJS.SOUND:         //sound loaded         handleLoadComplete();         break;         }}

Давайте просмотрим функции:

  • handleProgress - в этой функции вы сможете проследить за процентом хода загрузки при помощи этого параметра: event.loaded. Вы могли бы использовать его для создания индикатора выполнения хода задания.
  • handleComplete - эта функция вызывается сразу после загрузки всех файлов (на случай, если вы поместили в нее что-либо).
  • handleFileLoad – поскольку мы загружаем два вида файлов – изображения и звук – у нас имеется эта функция для обработки их по отдельности. Если это – изображение, то мы создаем растровое изображение и помещаем его в переменную (с тем же названием, что и у ID загруженного изображения), а затем вызываем функцию handleLoadComplete (которую напишем далее); если же это – звук, то мы тут же вызываем handleLoadComplete.

Теперь давайте обсудим только что упомянутую handleLoadComplete:

 function handleLoadComplete(event)  {totalLoaded++;if(manifest.length==totalLoaded){addTitleView();} }

В ней довольно все ясно: мы увеличиваем значение переменной totalLoaded (в которой содержится число ресурсов, загруженных на данный момент) и затем проверяем, совпадает ли число элементов в manifest с числом загруженных ресурсов, и если да, то переходим к экрану Main Menu (* Главное меню).

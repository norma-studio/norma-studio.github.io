JavaScript, благодаря своей популярности и последним улучшениям, быстро становится лучшим другом веб-разработчика. И как любой лучший друг, JavaScript держит свои обещания.

Сейчас это может показаться немного странным, но это правда. Большинство современных браузеров поддерживают то, что называется объект Promise. Объект Promise похож на функцию тем, что он представляет собой кусок кода или задачи, которая должна быть выполнена в определенный момент в будущем.

Вот как выглядит обещание.

var myPromise = new Promise(function (resolve, reject) {    // Task to carry out goes here.});

Здесь вы можете увидеть, что когда мы создаем обещание, то мы предаем ему один аргумент, который является функцией, содержащей код, который мы хотели бы выполнить в определенный момент в будущем. Вы так же могли заметить два аргумента, переданных в функцию обещания, resolve и reject. Это тоже функции и наша задача в том, чтобы объяснить объекту Promise, что ему нужно будет сделать после выполнения функции обещания. Вот как их следует использовать:

var myPromise = new Promise(function (resolve, reject) {    if (true) {        resolve("Hello Tuts+ fans!");    } else {        reject("Aww, didn\"t work.");    }});

Очевидно, что обещание всегда будет выполнено, так как выражение if всегда будет true. Но это только лишь в целях обучения, позже мы сделаем что-нибудь более реалистичное, но представьте себе замену true на фрагмент кода, в которым вы не уверены, что он на 100% выполнится успешно.

После того как мы создали обещание, как же нам его использовать? Нам нужно определить что из себя будут представлять функции resolve и reject. Это можно сделать, используя метод then у объекта обещания.

myPromise.then(function (result) {    // Resolve callback.    console.log(result); }, function (result) {    // Reject callback.    console.error(result);});

Так как наше if выражение всегда проходит проверку на true, то код, представленный выше, всегда выведет на консоль "Hello Tuts+ fans!". Причем это будет сделано немедленно. Это потому что код, внутри конструктора нашего объекта Promise синхронный, а значит он не будет ждать выполнения какой-либо операции. У него имеется вся необходимая информация, чтобы продолжить свое выполнение.

В действительности же всю мощь обещаний можно по достоинству оценить в асинхронных задачах - когда вы точно не знаете в какой момент времени обещание будет исполнено. Реальным примером асинхронной задачи является получение какого-либо ресурса, как например JSON файл, полученный через AJAX. Мы не знаем, как долго сервер будет отвечать, и возможно ответ даже не будет успешным. Давайте добавим немного AJAX в код нашего обещания.

var myPromise = new Promise(function (resolve, reject) {    // Standard AJAX request setup and load.    var request = new XMLHttpRequest();        // Request a user"s comment from our fake blog.    request.open("GET", "http://jsonplaceholder.typicode.com/posts/1");    // Set function to call when resource is loaded.    request.onload = function () {        if (request.status === 200) {            resolve(request.response);        } else {            reject("Page loaded, but status not OK.");        }    };    // Set function to call when loading fails.    request.onerror = function () {        reject("Aww, didn\"t work at all.");    }    request.send();});

Этот код представляет собой стандартный JavaScript для выполнения AJAX запроса. Мы запрашиваем ресурс, в этом случае JSON файл по определенному URL, и ждем ответа. Мы точно не знаем, когда ответ будет получен. Мы естественно не хотим прерывать выполнение скрипта, чтобы ждать ответа, так что же нам делать?

К счастью, можно обернуть этот код в обещание. Таким образом мы фактически говорим: "Эй, фрагмент когда, мне надо идти прямо сейчас, но я позже тебе перезвоню и скажу, когда тебе следует выполниться". "Ты обещаешь мне дать знать, как будешь готов?" Ну и конечно же код отвечает: "Да, несомненно. Я обещаю".

Важно обратить внимание во фрагменте кода, представленном выше, на вызов функций resolve и reject. Помните, что это наши способы показать обещанию, каким образом сообщить нам, что наш код выполнился успешно или неудачно. В противном случае мы этого никогда не узнаем.

Используя тот же самый код из нашего основного примера, мы можем увидеть каким образом AJAX запрос работает внутри объекта обещания.

// Tell our promise to execute its code// and tell us when it"s done.myPromise.then(function (result) {    // Prints received JSON to the console.    console.log(result);}, function (result) {    // Prints "Aww didn"t work" or    // "Page loaded, but status not OK."    console.error(result); });

Я знаю, что могу доверять тебе, myPromise.

Цепочка Обещаний

Сейчас вы возможно думаете, что обещания это всего лишь причудливые функции обратного вызова с приятным синтаксисом. Это от части правда, но если продолжить с нашим AJAX примером, допустим нам нужно выполнить еще несколько запросов, основываясь на результате последнего. Или что делать, если вам необходимо сперва нужно обработать JSON?

Попытка реализовать это через колбэки приведет к куче вложенных функций, каждая из которых становится трудно читаемой. К счастью в мире обещаний, вы можете составить цепочку вызовов из таких функций. Вот пример, где сначала мы получаем JSON с комментарием пользователя, а затем хотим убедиться, что весь комментарий написан в нижнем регистре.

myPromise    .then(function (result) {        // Once we receive JSON,        // turn it into a JSON object and return.        return JSON.parse(result);    })    .then(function (parsedJSON) {        // Once json has been parsed,        // get the email address and make it lowercase.        return parsedJSON.email.toLowerCase();    })    .then(function (emailAddress) {        // Once text has been made lowercase,        // print it to the console.        console.log(emailAddress);    }, function (err) {        // Something in the above chain went wrong?        // Print reject output.        console.error(err);    });

Можно заметить, что так как наш вызов был асинхронным, то можно так же составить цепочку из синхронных вызовов. Код в каждой функции resolve внутри then будет вызван. Вы также могли заметить, что здесь во всей цепочке определена только одна функция на случай ошибки. Мы расположили ее в конце цепочки, в виде reject функции в последнем then, и теперь любое обещание цепочки при вызове reject вызовет именно эту функцию.

Теперь, когда мы более уверенно чувствуем себя с обещаниями, создадим еще один объект в сочетании с тем, что был представлен выше. Мы создадим еще одно обещание, которое принимает наш адрес электронной почты в нижнем регистре и отправит письмо на этот адрес. Это всего лишь пример, иллюстрирующий асинхронность, это может быть что угодно, как например соединение с сервером, чтобы убедиться, что адрес не занесен в черный список или что пользователь залогинен. Мы должны передать адрес электронной почты обещанию, но обещание не принимает аргументов. Чтобы решить эту задачу, можно обернуть обещание в функцию, в которую уже затем передавать нужные параметры.

var sendEmail = function (emailAddress) {    return new Promise(function (resolve, reject) {        // Pretend to send an email        // or do something else asynchronous        setTimeout(function () {            resolve("Email sent to " + emailAddress);        }, 3000);    });};

Мы используем вызов setTimeout чтобы просто имитировать задачу, которая выполняется несколько секунд асинхронно.

Так как же нам использовать новую функцию по созданию обещаний? Так как каждая функция resolve, используемая в then должна возвращать функцию, то мы можем использовать такой же подход в наших синхронных задачах.

myPromise    .then(function (result) {        return JSON.parse(result);    })    .then(function (parsedJSON) {        return parsedJSON.email.toLowerCase();    })    .then(function (emailAddress) {        return sendEmail(emailAddress)    })    .then(function (result) {        // Outputs "Email sent to stuart@fakemail.biz"        console.log(result);    }, function (err) {        console.error(err);    });

Давайте рассмотрим все по порядку для того, чтобы обобщить. Наш исходный объект обещания myPromise делает запрос для получения JSON. После того как JSON будет получен, мы оборачиваем его в объект JavaScript и возвращаем полученное значение.

Как все будет готово, мы берем адрес электронной почты из объекта JSON и приводим его к нижнему регистру. Затем отправляем письмо на этот адрес, и опять же мы не знаем точно, когда эта задача будет завершена, но как отправка завершится, мы выведем соответствующее сообщение на консоль. И никаких вложенных функций.

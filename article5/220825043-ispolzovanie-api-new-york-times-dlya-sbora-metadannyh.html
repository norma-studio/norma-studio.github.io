Введение

На прошлой неделе я написал введение в парсинг веб-страниц для сбора метаданных, отметив, что невозможно спарсить сайт New York Times. Pywall Times блокирует ваши попытки собрать основные метаданные. Но есть способ обойти это, используя API Нью-Йорк Таймс.

Недавно я начал создавать сайт сообщества поверх платформы Yii, о котором я писал в «Программирование с Yii2: создание сообщества с комментариями, общим доступом и голосованием» (Envato Tuts +). Я хотел упростить добавление ссылок, связанных с контентом на сайте. В то время как людям легко вставлять URL-адреса в формы, но вот добавить название и источник уже будет занимать некоторое время.

Поэтому в сегодняшнем учебнике я собираюсь расширить код парсера, который я написал недавно, чтобы использовать API New York Times для сбора заголовков, когда добавляются ссылки Times.

Помните, я участвую в комментариях ниже, так что скажите мне, что вы думаете! Вы также можете связаться со мной в Twitter @lookahead_io.

The Times предлагает API в следующих категориях:

The Times использует LucyBot для управления своими документами API, и есть полезные FAQ:

Они даже показывают вам, как быстро получить ограничения на использование API (вам нужно подключить свой ключ):

Я изначально изо всех сил пытался понять документацию - это спецификация на основе параметров, а не руководство по программированию. Тем не менее, я поставил некоторые вопросы в качестве проблем на странице New York Times API GitHub, и они были быстро мне ответили.

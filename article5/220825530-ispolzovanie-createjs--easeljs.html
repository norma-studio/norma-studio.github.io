В данном руководстве мы будем рассматривать набор библиотек CreateJS. CreateJS – это набор библиотек JavaScript и библиотек для создания богатого, интерактивного опыта пользователя при помощи HTML5. Этот набор состоит из 4-х библиотек:

  • EaselJS – облегчает работу с canvas (* элемент HTML5, предназначенный для создания растрового двухмерного изображения при помощи JavaScript. Здесь и далее прим. пер.).
  • TweenJS – используется для твининга (* построение промежуточных отображений; плавный переход от одного значения к другому) свойств HTML5 и JavaScript
  • SoundJS – облегчает работу с HTML5 Audio
  • PreloadJS – позволяет вам управлять и согласовывать загрузку ресурсов.

Первая библиотека, которую мы рассмотрим, – это EaselJs. Давайте приступим!

Как работать с EaselJS

EaselJS – библиотека для работы с canvas HTML5. Она состоит из полного иерархического дисплейного списка (* в компьютерной графике - набор векторов, используемых для построения изображения на экране), базовой модели взаимодействия и вспомогательных классов для значительного облегчения работы с Canvas. Начать работу с EaselJS очень просто. Вам необходимо будет определить элемент canvas в вашем HTML и сослаться на него в вашем JavaScript. EaselJS использует концепцию Stage (* сцены), которая является Container высшего уровня для дисплейного списка (сценой будет ваш элемент canvas). Ниже представлен фрагмент HTML и соответствующий код JavaScript, необходимый для настройки EaselJS.

<canvas id="testCanvas" width="600" height="300"></canvas>

Для того, чтобы сослаться на canvas выше, вы бы использовали следующий код:

 var canvas = document.getElementById("testCanvas"); var stage = new createjs.Stage(canvas);

Рисование при помощи классов Graphics и Shape

У EaselJS имеется класс Graphics, который предоставляет легкий в использовании API для создания инструкций векторного рисунка и прорисовывания их на указанный контекст. Команды очень похожи на те, что используются в обычном Canvas HTML5, а также в EaselJs имеются некоторые собственные команды для работы с Canvas. Если вы не знакомы с HTML5 и командами для рисования, возможно, вы захотите посмотреть недавно вышедший курс Canvas Essentials. Обычно вы не будете использовать сам класс Graphics, а скорее обратитесь к нему при помощи класса Shape. Ниже представлен JSFiddle (* онлайн редактор кода для тестирования JavaScript, HTML и CSS), в котором используется тот самый класс Shape для рисования некоторых фигур.

Как вы можете увидеть в коде выше, команды можно прицеплять к друг другу: ShapeObject.graphics.setStrokeStyle(1).beginStroke("rgba(0,0,0, и т.д. Также вы могли заметить вызов stage.addChild() внутри функций. Каждый раз, когда вы хотите прорисовать что-либо на canvas, вы должны вызывать stage.addChild(). Внутри функции drawShapes() имеется вызов stage.update(). Для того чтобы увидеть сделанные изменения, вы должны вызвать stage.update() после внесения каждого из них на canvas. Этой необходимости можно избежать при помощи класса Ticker. Ticker предоставляет централизованный шаг или ритм, рассылаемый по заданному интервалу. Обработчики событий могут подписываться на событие tick, чтобы быть получить оповещение о том, что заданный промежуток времени прошел. Ниже показано, как бы вы использовали класс Ticker для автоматического обновления сцены.

 createjs.Ticker.addEventListener("tick", handleTick); function handleTick(event) {   stage.update() }

Затем мы рандомизируем (* придаем случайный характер) shapes position (* положение фигур), scale (* масштаб), rotation (* поворот) и alpha (* в компьютерной графике - четвёртый компонент цвета, используемый для контроля смешивания цветов с фоном или нижележащим объектом. Sprite и SpriteSheets (* спрайт; легко перемещаемый по экрану растровый графический элемент (например, пуля или ракета), часто реализуемый с помощью аппаратных средств)

Sprite отображает кадр или последовательность кадров (на подобие анимации) из образца SpriteSheet. Sprite sheet – это серия изображений (обычно кадров анимации), скомбинированных в единое изображение. Например, анимация, состоящая из 8 изображений размером 100x100, может быть сгруппирована в sprite sheet размером 400x200 (шириной в 4 кадра и высотой в 2). Вы можете отобразить индивидуальные кадры, показать их в виде анимации и даже проиграть серию анимаций. Для инициализации образца Sprite вы передаете SpriteSheet и, как необязательные параметры, число кадров или нужный вам тип анимации, например, new createjs.Sprite(spriteSheet,"run"). Данные, передаваемые в конструктор SpriteSheet, определяют три важных части информации:

  • Изображение или изображения, которые необходимо использовать.
  • Расположение отдельных кадров изображений. Эти данные могут быть представлены двумя способами: в виде регулярной сетки (решетки) (* двумерная сетка в виде множества точек, расположенных с определённым одинаковым шагом в рабочем окне) последовательных, равновеликих кадров или в виде кадров разных размеров, свойства которых заданы индивидуально и которые расположены расположены нерегулярным (непоследовательным) способом.
  • Подобно предыдущим данным анимации могут быть представлены двумя способами: в виде серии последовательных кадров, начало и конец которой определяется начальным и конечным кадром [0,3], или же в виде списка кадров [0,1,2,3].

Ниже приведен пример кода, при помощи которого происходит инициализация спрайта «персонажа".

var data = new createjs.SpriteSheet({"images": ["path/to/image.png"],"frames": {"regX": 0, "height": 96, "count": 10, "regY": 0, "width": 75},"animations": {"walk": [0, 9]}});character = new createjs.Sprite(data, "walk");character.play();

Cache()

При использовании метода cache() объекта отображения он прорисовывается на новый canvas, который затем используется для дальнейших прорисовок. Для сложного контента, который изменяется редко (например, Container со многими дочерними элементами, которые не двигаются, или Shape на основе комплексного вектора), благодаря этому методу может быть достигнуто более быстрое отображение, поскольку контент не должен заново отображаться при каждом шаге.

Помещенный в кэш объект отображения можно свободно двигать, вращать, убирать и т. д., однако при изменении его контента вам необходимо будет вручную обновить кэш при помощи повторного вызова updateCache() или cache(). Вам необходимо задать область для кеширования через параметры x, y, w, и h. За счет нее определяется прямоугольник, который будет отображен и помещен в кэш на основании координат объекта отображения. Как правило кеширование не стоит использовать для класса Bitmap, поскольку из-за этого может понизиться производительность, однако если вы хотите использовать фильтр для Bitmap, то его следует поместить в кэш.

Из-за ограничений размера для подключений на JSFiddle я не буду предоставлять демоверсию для этого метода, однако дам вам ссылку на демоверсию на сайте CreateJs.

Filters (* фильтры)

Класс Filter – это базовый класс, от которого должны унаследовать свойства все остальные фильтры. Фильтры должны быть применены к объектам, которые были помещены в кэш при помощи метода cache. Если объект изменяется, вам необходимо будет поместить его в кэш снова или использовать updateCache(). Обратите внимание, что фильтры должны быть применены перед помещением в кэш. EaselJS предоставляет ряд предварительно созданных фильтров. Обратите внимание, что отдельные фильтры не скомпилированы в минифицированную версию EaselJS. Для того, чтобы их использовать, вам необходимо подключить их вручную в HTML.

  • AlphaMapFilter : подключает шкалу яркости (* полутоновое изображение, представляющее только плавный переход от чёрного через серый к белому и позволяющее оценить количество одновременно отображаемых оттенков серого цвета) к каналу alpha объекта отображения
  • AlphaMapFilter: подключает канал alpha изображения к каналу alpha объекта отображения
  • BlurFilter: применяет вертикальное и горизонтальное размытие к объекта отображения
  • ColorFilter: трансформирует цвет объекта отображения
  • ColorMatrixFilter: трансформирует изображение при помощи ColorMatrix

В демоверсии ниже используется Color Filter; при клике по сцене благодаря ему происходит удаление зеленого канала со сцены. Color Filter имеет следующие параметры:

ColorFilter( [redMultiplier=1] [greenMultiplier=1] [blueMultiplier=1] [alphaMultiplier=1] [redOffset=0] [greenOffset=0] [blueOffset=0] [alphaOffset=0])

Предполагается, что Multipliers red, green, blue и alpha будут числами от 0 до 1, а Offset red, green, blue и alpha – числами от 255 до 255.

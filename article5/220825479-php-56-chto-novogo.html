Прошло много времени в ожиданиях, и вот наконец вышел новый релиз PHP. Он предлагает несколько новых возможностей и улучшений, в то время как некоторые другие возможности были удалены или помечены как устаревшие.

Давайте углубимся и рассмотрим более детально, что нам предлагают последние версии.

Обратно несовместимые изменения

В этой части мы перечислим обратно несовместимые изменения, несмотря на то что большая часть кода PHP5 будет корректно работать и в PHP 5.6 без каких-либо изменений.

json_decode

Согласно спецификации JSON, json_decode() будет немедленно удалять все варианты JSON литералов не нижнего регистра, такие как как true, false и значение null. Так же будет соответственно установлена json_last_error().

GMP Ресурсы

Если вы не уверены, какая максимальная точность у вас в PHP, то я советую вам взглянуть на эту статью на PHP.NET (EN). В PHP 5.6 GMP ресурсы представляют собой объекты. Вам не нужно делать никаких изменений в вашем существующем коде, до тех пор пока вы явно не проверяете его с помощью is_resource().

Mcrypt

Все функции Mcrypt, которые ожидают ключи или IV, теперь не будут принимать ключи или IV с неверным размером. Сюда входят такие функции как mcrypt_encrypt(), mcrypt_decrypt(), mcrypt_cbc(), mcrypt_cfb(), mcrypt_ecb(), mcrypt_generic(), и mcrypt_ofb().

Значения массива и перезапись

До PHP 5.6 когда вы объявляли массив в свойстве класса с явными или неявными ключами, то значение массива перезаписывалось, когда явный ключ был такой же как и не явный.

class helloWorld {      const ONE = 1;public $array = [self::ONE => "Eclipse","Sublime","PHPStorm",];}$hello = new helloWorld();print_r($hello);// Before PHP 5.6array("Sublime", "PHPStorm")// PHP 5.6array("Eclipse", "Sublime", "PHPStorm")

Новые возможности в PHP 5.6

Константы в виде скалярных выражений

С выходом PHP 5.6 теперь стало возможно использовать скалярные выражения с использованием как чисел так и строк. В предыдущих версиях PHP, ожидалось что значением константы будет статическое определение.

const ONE = 1;// Scalar Expression in constantconst TWO = ONE * 2;class helloWorld {    // Scalar Expression in Propertyconst THREE = TWO + 1;// Scalar Expression in Methods    public hello f($a = ONE + self::THREE) {        return $a;    }}echo (new helloWorld)->hello()."\n";

Функции с переменным числом аргументов "..."

Раньше мы использовали func_get_args() для получения всех доступных аргументов при вызове функции, но с PHP 5.6 ее можно удалить, так как можно легко достичь того же с помощью оператора ...

function myTools($name, ...$tools) {    echo "Name:". $name."<br />";echo "My Tool Count:". count(tools);}myTools("Avinash", "Eclipse");// Output:// Name: Avinash// My Tool Count: 1myTools("Avinash", "Eclipse", "Sublime");// Output:// Name: Avinash// My Tool Count: 2myTools("Avinash", "Eclipse", "Sublime", "PHPStorm");// Output:// Name: Avinash// My Tool Count: 3

Распаковка аргументов

Можно использовать тот же оператор (...) для распаковки любого аргумента. 

function myTools($name, $tool1, $tool2, $tool3) {    echo "Name:". $name."<br />";echo "Tool1:", $tool1."<br />";echo "Tool2:", $tool2."<br />";echo "Tool3:", $tool3;}$myTools = ["Eclipse", "Sublime", "PHPStorm"];myTools("Avinash", ...$myTools);// Output:// Name: Avinash// Tool1: Eclipse// Tool1: Sublime// Tool1: PHPStorm

** Сокращение

Был добавлен оператор ** для возведения в степень. Вот так легко можно его использовать.

echo 2 ** 3;echo "<br/>";$a=2;$a **= 3;echo $a;// Output// 8// 8

Обратите внимание, что в этом операторе порядок оперном имеет значение. Рассмотрим этот пример, чтобы более подробно это прояснить:

echo 2 ** 2 ** 4;

Вы возможно ожидаете, что результат будет 256, так как группировка ожидается такой (2 ** 2) ** 4, но не в нашем случае. Реальный результат будет 65,536, так как операнды группируются справа налево и выражение будет распарсено так 2 ** (2 ** 4).

phpdbg

В PHP 5.6 был добавлен интерактивный отладкчик phpdbg. Документация по нему доступна на официальном сайте phpdbg.

Отладчик phpdbg реализован в виде SAPI модуля.

__debugInfo()

Новый магический метод в PHP 5.6 позволяет вам менять свойства и значения объекта во время его вывода функцией var_dump().

class demoDebugInfo() {    private $val;public function __construct($val) {$this->val = $val;}public function __debuginfo() {return ["iamchanged" => $this->val + 3;];}}$debugInfo = new demoDebugInfo(50);// We will get altered array which is returned by __debuginfo megic method.var_dump($debugInfo);

Кодировка по умолчанию

Кодировка по умолчанию для функций htmlentities(), html_entity_decode() и htmlspecialchars() может быть установлена с помощью default_charset().

Загрузка больших файлов

Можно загружать файлы размером больше чем 2 Гб.

php://input можно использовать повторно.

php://input можно использовать столько раз, сколько вам нужно для чтения данных. Эта возможность обеспечивает значительное сокращение в использовании памяти по сравнению с чтением данных из POST.

use function и use const

Оператор use теперь поддерживает использование функций и констант. Это можно достичь используя const, use и function. Ранее этот оператор был ограничен лишь использованием с классами.

Устаревшие возможности

Вызов из неподходящего контекста

Если вы пытаетесь получить доступ к нестатическому методу статично, или наоборот статичный метод используется в контексте объекта, то будет сгенерирована ошибка E_DEPRECATED.

class helloWorld() {    public static hiWorld () {}}$hello = new helloWorld();$hello->hiWorld();// Before PHP 5.6 E_STRICT error// From PHP 5.6E_DEPRECATED error

$HTTP_RAW_POST_DATA

$HTTP_RAW_POST_DATA теперь считается устаревшим. Вместо этого следует использовать php://input.

Настройки кодировки

После включения возможности конфигурации кодировки с помощью функции default_charset(), соответствующая опция в iconv и mbstring считается устаревшей.

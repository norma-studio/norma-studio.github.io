Как и большинство других языков, Node предоставляет отличные инструменты для отладки, которые легко обнаруживают и исправляют баги в коде. Я всегда выступаю за использование отладчика, потому что лично я считаю, что использование отладчиков действительно устраняет необходимость в каких-либо догадках и улучшает нас как разработчиков.

Это руководство предназначено для разработчиков и администраторов, которые уже работают с Node. Оно предполагает фундаментальное понимание языка на практическом уровне.

Использование отладчика

Node.js включает полнофункциональную утилиту для отладки вне процесса, доступную через простой TCP-протокол и встроенный клиент отладки.

Например, чтобы использовать отладчик для отладки файла с именем script.js, вы можете просто вызвать node с помощью флага debug следующим образом:

$ node debug script.js< debugger listening on port 5858connecting... okdebug>

Контрольные точки

Теперь, когда вы запустили сеанс отладки, любое место вашего сценария, от которого вы вызываете debugger, будет точкой останова для отладчика.

Так, например, добавим отладчик к script.js:

foo = 2;setTimeout(() => {  debugger;  console.log("bugger");}, 1000);console.log("de");

Теперь, если мы запустим этот скрипт, отладчик будет вызван на нашу контрольную точку, и мы можем управлять элементом управления сценария, используя команды cont или next (сокращено c или n).

Мы можем приостановить выполнение скрипта в любое время, используя p.

$ node debug script.js< debugger listening on port 5858connecting... okbreak in /home/tom/web/envatodebug/myscript.js:1  1 foo = 5;  2 setTimeout(() => {  3   debugger;debug> cont< debreak in /home/tom/web/envatodebug/myscript.js:3  1 foo = 5;  2 setTimeout(() => {  3   debugger;  4   console.log("bugger");  5 }, 1000);debug> nextbreak in /home/tom/web/envatodebug/myscript.js:4  2 setTimeout(() => {  3   debugger;  4   console.log("bugger");  5 }, 1000);  6 console.log("de");debug> next< buggerbreak in /home/tom/web/envatodebug/myscript.js:5  3   debugger;  4   console.log("bugger");  5 }, 1000);  6 console.log("de");  7debug> quit

REPL

$ node debug script.js< debugger listening on port 5858connecting... okdebug> replPress Ctrl + C to leave debug repl> foo2> 2+24

Read-Eval-Print-Loop отладчика позволяет вам вводить код интерактивно во время выполнения и, таким образом, получать доступ к состоянию приложения и всем его переменным и методам в момент прерывания выполнения. Это очень мощный инструмент, который можно использовать для быстрой инспекции вашего приложения.

В общем, REPL доступен автономно и как часть отладчика, и он позволяет вам выполнять JavaScript в интерактивном режиме. Например, просто введите node без параметров, и вам будет предоставлен интерфейс REPL, в который вы можете записать код и увидеть результат.

Пошаговое выполнение и выход

Ранее я упоминал команды cont и next (c и n), которые позволяют нам продолжать выполнение кода после достижения точки останова. В дополнение к этому, когда мы проходим код, мы также можем перейти к методу или выйти в его родительскую область.

Используйте команды step для входа и out, чтобы выйти, или сокращено s и o.

Обратная трассировка

Используйте backtrace или bt, чтобы получить вывод backtrace для текущего фрейма выполнения.

Перезапуск

Используйте команду restart или r для перезапуска скрипта с начала выполнения.

Альтернативные способы подключения к отладчику

Продвинутые пользователи могут получить доступ к отладчику, также запуская Node.js с флагом командной строки --debug, или сигнализируя существующий процесс Node.js с помощью SIGUSR1.

Как только процесс был установлен в режим отладки таким образом, его можно затем подключить с помощью отладчика Node.js либо с использованием pid запущенного процесса, либо через ссылку URI (например, localhost:port) для подключения прослушивающего отладчика:

  • node debug -p <pid> подключается к процессу через pid.
  • node debug <URI> подключается к процессу через URI, такой как localhost:5858.

Использование инспектора узлов

В дополнение к инструменту отладки CLI, Node Inspector также предоставляет инспектор GUI внутри веб-браузера (в настоящее время поддерживающий только Chrome и Opera).

Чтобы использовать отладчик, просто установите его таким образом:

npm install -g node-inspector

Теперь, когда у нас установлен Node inspector, мы можем отладить наш script.js с помощью:

node-debug script.js

Теперь ваша оболочка выдаст следующее и, вероятно, откроет веб-браузер для URL-адреса, если вы установили Chrome или Opera в качестве своего браузера по умолчанию.

Node Inspector is now available from https://127.0.0.1:8080/?ws=127.0.0.1:8080&port=5858 Debugging `script.js`  Debugger listening on port 5858

В своем веб-браузере вы теперь сможете отлаживать свое приложение в аналогичной среде с пакетом инструментов разработчика. Установка точек останова и просмотр кода теперь интегрированы в представление вашего браузера. Наслаждайтесь!

Вывод

Отладка не должна быть кошмаром, и она не должна быть стрессом.

Установка точек останова и пошаговое выполнение кода в Node довольны просты. Это очень похоже на опыт Ruby, и если вы пытаетесь понять приложение, которое попало вам в руки, открытие приложения в режиме отладки и приостановка выполнения - отличный способ научиться быстрому таймфрейму.

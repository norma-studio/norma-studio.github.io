В первой части этой серии «Пересмотр альтернатив для социальных сетей с открытым исходным кодом» я дал краткий обзор нескольких открытых сетевых платформ и клиентов с открытым исходным кодом. В этой части я погружусь в установку GNU Social и мы обсудим то, что, по моему мнению, будет иметь решающее значение для конкуренции в доминирующем мире Facebook и Twitter.

Почему GNU Social? Я чувствовал, что это самая простая система для установки и использования. Она также написана на PHP, который я считаю самым доступным языком для опен-сорса. Хотя она ограничивает себя как клон Twitter, я думаю, что это помогло сохранить все ее основные функции и веб-интерфейс пользователя относительно чистым. Тем не менее, веб-сайт GNU Social не работал в течение нескольких дней, пока я работал над этим уроком; состояние распределенной сети порой остается неясным.

Установка GNU Social

Вы можете установить GNU Social на любую систему на основе LAMP. Я опубликовал свой общий справочник по установке стека Ubuntu LAMP.

Войдите на свой сервер:

ssh root@yourserver.com

Установите все необходимые компоненты (Curl и Git также необходимы для этого урока):

sudo locale-gen en_US en_US.UTF-8sudo dpkg-reconfigure locales sudo apt-get update -qsudo apt-get dist-upgrade -ysudo apt-get install -y pound varnish php5 rsync htop emacs23-nox apache2 php5-mysql dtrx mysql-server exim4 sudo apt-get install php5-curl git

Создайте каталог для сайта и установите разрешения:

mkdir /var/www/social -pchown www-data:www-data /var/www/socialcd /var/www/social

Клонируем репозиторий:

git clone git://gitorious.org/social/mainline.git .

Войдите в MySQL с вашим рутовым паролем базы данных:

mysql -u root -p

Создайте новую базу данных и установите разрешения:

create database social;grant all privileges on social.* TO "root"@"localhost" identified by "-your-pwd-";flush privileges;exit;

Создайте сайт Apache:

nano /etc/apache2/sites-available/social.conf

Вставьте в конфигурацию Apache ниже, адаптированную под ваш сервер:

<VirtualHost *:80> ServerName social.yourdomain.com DocumentRoot /var/www/social DirectoryIndex index.php <Directory /var/www/social/> AllowOverride All Order Deny,Allow Allow from all </Directory></VirtualHost>

Включите сайт, отключите сайт по умолчанию и перезапустите Apache:

sudo a2ensite social.confsudo a2dissite 000-default.confsudo service apache2 reload

Вам также необходимо настроить несколько каталогов для доступа на запись:

chmod a+w /var/www/social/avatar/chmod a+w /var/www/social/background/chmod a+w /var/www/social/file/

Чтобы активировать красивые URL, мы должны перенести файл .htaccess из образца и включить mod_rewrite:

mv htaccess.sample .htaccesssudo a2enmod rewritesudo service apache2 restart

Перейдите на страницу установки и введите свое имя и настройки:

http://social.yourdomain.com/install.php

Установка пользовательской темы

Давайте установим тему Neo Quitter с открытым исходным кодом, которая напоминает Twitter:

cd /var/www/social/themegit clone https://github.com/hannesmannerheim/neo-quitter

Отредактируйте файл конфигурации и добавьте запись для темы Neo Quitter:

cd /var/www/socialsudo nano config.php

Он должен выглядеть так:

<?phpif (!defined("GNUSOCIAL")) { exit(1); }$config["site"]["name"] = "Social";$config["site"]["server"] = "social.lookahead.io";$config["site"]["path"] = false; $config["site"]["ssl"] = "never"; $config["site"]["fancy"] = true;$config["db"]["database"] = "mysqli://root:your-pwd@localhost/social";$config["db"]["type"] = "mysql";$config["site"]["profile"] = "public";

$config["site"]["theme"] = "neo-quitter";

Экспериментирование с федеративной сетью

Затем я хотел поэкспериментировать с объединенными сетевыми функциями GNU Social. Я перескочил на Quitter.is, социальный сайт GNU, открытый для публики, и последовал за первым человеком, которого я там нашел.

На своем собственном сайте вы должны посетить страницу своего профиля, а затем нажать «Следующий + удаленный» в правом столбце (это не совсем интуитивно понятно):

Чтобы следовать за удаленным пользователем, вы создаете адрес, объединяющий имя удаленного пользователя с доменом удаленного сетевого сайта, например. username@quitter.is.

Хотя производительность относительно медленная (этот Quitter находится в Исландии), мы смогли обменять @replies, и моя учетная запись GNU Social появилась в списке последователей Quitter.is. Когда я вошел в систему на следующий день, их обновления появились на моей установке GNU Social:

Конкурируем с Facebook и Twitter

Основная задача

Вот некоторые из качеств, которые, как я думаю, важны для системы социальных сетей с открытым исходным кодом для набора популярности:

  • Простая установка для низкоквалифицированных разработчиков. Например, для тех, у кого есть PHP и базовые знания Linux. Следует приблизиться к установке WordPress.
  • Одиночные и многопользовательские опции. В идеале платформа должна поддерживать запуск установки только для одного пользователя (установщика) или как сеть, которая поддерживает сообщество многих пользователей.
  • Платный хостинг, чтобы позволить заинтересованным нетехническим пользователям тоже участвовать. Например, мне нравится, как платформа блогов Google Ghost предлагает платный SaaS вариант.
  • Элегантный пользовательский интерфейс постинга (текст, фотографии, видео) и поддержка поиска метатегов из URL-адресов. Это одна из самых важных и упущенных функций в сегодняшних предложениях с открытым исходным кодом.
  • Простой пользовательский интерфейс. Интерфейс должен быть таким же простым, как в Facebook и Twitter.
  • Хорошая производительность. Обычные операции, например публикация, загрузка фотографий, просмотр, просмотр временной шкалы и получение обновлений временной шкалы должны быть быстрыми.
  • Публикация в Twitter и Facebook. Платформа должна позволять пользователям зеркально отражать свои локальные сообщения в своих существующих сетях автоматическим или полуавтоматическим способом.
  • Поддержка Тем. Пользователи должны иметь возможность персонализировать внешний вид сети.
  • Стратегия резервного копирования. Платформа должна включать встроенную возможность резервного копирования, в идеале с некоторыми облачными опциями, такими как S3.
  • Интеграция аналитики. Пользователям должно быть легко получить данные о деятельности своих каналов.
  • Хорошая архитектура кода, например. MVC и/или использование фреймворка.
  • Отличная документация по кодовой базе.
  • Полуавтоматическое обновление. Очень важно сделать цикл обновления простым для людей, чтобы обновлять кодовую базу.

Расширенные возможности

Кроме того, многие аспекты того, что делает Facebook и Twitter успешными и доминирующими, являются утонченными или невидимыми для нас. Например, вот несколько важных:

  • Высокая производительность по всей сети
  • Высокая пропускная способность для загрузки изображений (и, возможно, видео)
  • Отказоустойчивость к использованию
  • Никогда не теряют данные

Хотя распределенный характер сетей с открытым исходным кодом может помочь с ограничением ущерба от отказа одного узла, он также увеличивает сложность решения проблем производительности и управления пользовательским интерфейсом, поскольку сеть взаимодействует с множеством установленных узлов.

Мост к существующим сетям

Чтобы ускорить принятие, для этих сетей также может быть важно импортировать фиды из Twitter (и, возможно, Facebook). Например, если вы переключитесь с Twitter на свою социальную сеть, вы можете стать первым в вашей группе друзей. Вероятно, вы захотите продолжить получать обновления и уведомления, получать последующие запросы и зеркальные ответы из старой сети на своем сайте - без необходимости возвращаться в Twitter.

Текущая технология API Twitter (User Streams) поддерживает это как техническую возможность для отдельных пользователей, но для этого требуется разрешение на поддержку группы пользователей (потоки сайтов). См. Работа с API Twitter (Tuts +). Я предполагаю, что маловероятно, что Twitter предоставит разрешение Site Stream для такого взаимодействия.

Оценка GNU Social

Базовая оценка

GNU Social работает очень хорошо, но оставляет значительные возможности для улучшения. Она отлично справляется со своим узким набором функций и неплохо их реализует.

Установка кодовой базы может быть немного улучшена. В идеале должны быть пакеты, которые могут быть быстрее установлены. В конце концов, я хотел бы увидеть руководство по установке и обслуживанию для нетехнических пользователей.

Существует множество сетей, где вы можете зарегистрироваться для бесплатной учетной записи и попробовать свою платформу. В будущем было бы неплохо видеть, что провайдеры предлагают устанавливать и поддерживать сети для людей на платной основе.

Хотя база кода чистая, она явно не использует никакой фреймворк.

GNU Social может работать как для отдельных пользователей, так и для сообщества. Поддержка OStatus, похоже, работает хорошо. И есть поддержка тем.

Интерфейс постинга является чистым, но не предлагает поиска для URL-адресов. Кажется, что нет никакой поддержки для мета-тегов, Open Graph-тегов или карточек Twitter.

Опыт конечных пользователей относительно прост, но он все равно может быть улучшен. Например, следование за удаленными пользователями не было доступно из домашней временной шкалы. Мне пришлось перейти на страницу моего профиля, чтобы следить за удаленными пользователями.

Производительность была болезненной, когда дело доходило до подключения к удаленным сетям - возможно, потому, что я был подключен к серверу на базе Исландии. Вероятно, важно, чтобы все эти операции были поставлены в очередь в качестве фоновых задач.

Зеркалирования для Twitter и Facebook, создание резервных копий и аналитики ещё находятся в самом зачатке.

Заключение

Энтузиасты PHP могут использовать GNU Social, запустить свои собственные сайты и внести свой вклад в кодовую базу. Я также чувствую, что команда разработчиков с открытым исходным кодом, стоящая за GNU Social, может добиться больших успехов, сохранив при этом набор функций и сосредоточив внимание на полировке того, что у них уже есть. Обеспечение супер-надёжной, хорошо продуманной, хорошо документированной кодовой базы, которая может воссоздать Twitter в федеративном режиме, будет значительным достижением, и они сейчас на этом пути.

Хотя GNU Social построена для поддержки OStatus, в рабочей группе W3C Social Web работает активная рабочая группа; вы можете следить за их работой в Twitter @socialwebwg.
